# Product Requirements Prompt (PRP) — 🔘 Pending

## 1. Project Context & Objectives
**Bug Fix Operation:** The initial interactive selection implementation introduced critical UX failures due to DOM stacking conflicts, coordinate space mismatches, and incorrect keyboard event mapping. 
* Note selection is being intercepted by overlapping transparent parent containers.
* Marquee intersection math is failing due to comparing relative state coordinates against absolute viewport DOM coordinates.
* Arrow key navigation is mapped to Left/Right instead of strictly Up/Down.

## 2. Technical Decisions & Dependencies
* **Hitbox Precision Isolation:** The parent `.notation-note-container` must NOT have `pointer-events: auto`. It must remain transparent to clicks. Only the explicitly rendered SMuFL text node `div`s (notehead and accidental) will have `pointer-events: auto` and the `onClick` handler.
* **Unified Coordinate Space:** To ensure marquee math is flawless, attach a `ref` to the DOM element representing the drawn marquee box. On `mouseUp`, calculate the intersection using `marqueeRef.current.getBoundingClientRect()` against `note.getBoundingClientRect()`. This guarantees both rects are in the exact same viewport-relative coordinate system.
* **Canvas Background Deselection:** Clicking the empty canvas background must clear the `selectedPitches` set entirely.

## 3. Implementation Phases

### Phase 1: Hitbox Precision & Click Deselection — 🔘 Pending
* **Objective:** Ensure clicks reliably hit the exact note intended, and clicking the background clears the selection.
* **Instructions:**
    1. In `NotationCanvas.tsx`, remove `pointer-events: auto` and the `onClick` handler from `.notation-note-container`.
    2. Add `pointer-events: 'auto'` and the `onClick` handler directly to the inner `div` rendering the notehead (`SMuFL.noteheadWhole`), AND the inner `div` rendering the accidental. (Pass the MIDI note data to these handlers).
    3. Add an `onClick` handler to the root `.notation-canvas-container` that clears `selectedPitches`. **Crucial:** In the notehead/accidental `onClick` handlers, you MUST call `e.stopPropagation()` to prevent the click from bubbling up to the canvas and immediately clearing the selection.

### Phase 2: Viewport-Relative Marquee Math — 🔘 Pending
* **Objective:** Fix the marquee bounding box intersection logic.
* **Instructions:**
    1. Ensure the visual marquee box `div` has a `ref` (e.g., `marqueeRef`).
    2. In the `onMouseUp` handler, do NOT use the React state `x, y, width, height` for the intersection math.
    3. Instead, extract `const selectionBox = marqueeRef.current.getBoundingClientRect()`.
    4. Iterate over `document.querySelectorAll('[data-midi-note]')` (ensure you add this attribute to the *inner* notehead divs now, not the parent wrapper).
    5. For each node, `const noteBox = node.getBoundingClientRect()`. 
    6. Apply strict AABB logic: `if (!(noteBox.left > selectionBox.right || noteBox.right < selectionBox.left || noteBox.top > selectionBox.bottom || noteBox.bottom < selectionBox.top)) { add to selected set }`.
* **AGENT INSTRUCTION:** Set Phase status to 🔵 Active. Audit and rewrite the intersection math. Set status to ✅ Completed.

### Phase 3: Vertical-Only Navigation Routing — 🔘 Pending
* **Objective:** Restrict selection navigation strictly to the vertical axis (Up/Down arrows) and disable horizontal navigation placeholders.
* **Instructions:**
    1. In the `keydown` listener, locate the bare arrow key handlers (no modifier keys).
    2. If `e.key === 'ArrowLeft' || e.key === 'ArrowRight'`, do nothing (`return;`).
    3. If `e.key === 'ArrowUp'`:
        * Get the maximum pitch currently in `selectedPitches`.
        * Find the smallest pitch in `activeNotes.current` that is strictly *greater* than that maximum.
        * If one exists, set `selectedPitches` to contain *only* that new pitch.
    4. If `e.key === 'ArrowDown'`:
        * Get the minimum pitch currently in `selectedPitches`.
        * Find the largest pitch in `activeNotes.current` that is strictly *less* than that minimum.
        * If one exists, set `selectedPitches` to contain *only* that new pitch.
    5. Ensure `e.preventDefault()` is called for Up/Down arrows to prevent window scrolling.
* **TDD Checkpoint:**
    * **Test Case 1:** Given `activeNotes` [60, 64, 67] and `selectedPitches` [64]. Trigger `ArrowUp`. Assert `selectedPitches` becomes [67]. Trigger `ArrowDown` twice. Assert `selectedPitches` becomes [60].
    * **AGENT INSTRUCTION:** Set Phase status to 🔵 Active. Implement and test. Once PASS, set status to ✅ Completed.
