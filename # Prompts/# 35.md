# Product Requirements Prompt (PRP) — 🔘 Pending

## 1. Project Context & Objectives
The previous layout updates failed because of a mathematical disconnect inside `src/components/NotationCanvas.tsx`. 

The gap calculation (`rightBaseX`) only provisioned `1.2 * staffSpace` of room. However, the accidental rendering logic (`processAccColumns`) inherently pushes accidentals `1.5 * staffSpace` to the left, and even further if multiple accidental columns are generated. This guarantees a collision where the Right Stack's accidentals bleed backward into the Left Stack.

Your objective is to rewrite the gap calculation inside `NotationCanvas.tsx` so that it pre-calculates the exact leftward reach of the Right Stack's accidentals, and uses that dynamic value to establish `rightBaseX`.

## 2. Technical Decisions & Dependencies
**Dynamic Reach Pre-Calculation:**
Before `rightBaseX` can be set, the algorithm must figure out how many columns of accidentals the Right Stack requires.
* If 0 columns: `Reach = 0`.
* If 1 column: `Reach = 1.5 * staffSpace` (from `ACC_BASE_OFFSET`).
* If 2 columns: `Reach = (1.5 + 1.2) * staffSpace`.
* **Formula:** `MaxReachPx = (1.5 + ((numColumns - 1) * 1.2)) * staffSpace`.

The new `rightBaseX` must be: `leftMaxX + MaxReachPx + (0.2 * staffSpace)`.
