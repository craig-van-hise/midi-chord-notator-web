# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
We are expanding the $O(1)$ chromatic root spelling engine to handle three new specific sonority rules based on user requirements:

1. **m2 (Minor 2nd):** All isolated minor 2nd intervals (Pitch Class Set decimal `3`) must be assumed sharp when falling out of key context. They are currently evaluating as flat.
2. **Phrygian Pattern:** Any chord with `"phrygian"` in the `chord_type` must be assumed sharp across all out-of-key root degrees.
3. **Dorian Pattern:** Any chord with `"dorian"` in the `chord_type` must utilize the exact same routing as the Minor Pattern established previously (`#1, b3, #4, #5, b7`).

## 2. Technical Decisions & Dependencies
* **m2 (Decimal 3) Fix:** The cardinality-2 fallback logic currently uses an array (`check14679e`) to route specific decimals to the flat-side array (index `0`). Decimal `3` is currently in this array. By simply removing `3` from the array, the existing ternary operator will cleanly fall back to the sharp-side array (index `1`).
* **Dorian Integration:** We will update the temporary extraction script to include `entry.chord_type.includes("dorian")` in the `isMinor` boolean logic. This seamlessly folds Dorian decimals into the existing `MINOR_PATTERN_DECIMALS` set, requiring zero new runtime routing logic.
* **Phrygian Extraction:** We will add an `isPhrygian` boolean to the extraction script (`entry.chord_type.includes("phrygian")`) and hardcode a new `PHRYGIAN_PATTERN_DECIMALS` set for $O(1)$ routing.
