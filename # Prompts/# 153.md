# Product Requirements Prompt (PRP) — 🔘 Pending

## 1. Project Context & Objectives
**Context:** The Grand Staff MIDI Notator currently derives its enharmonic chord spellings deterministically from a static 56,000+ entry Pitch Class Set Look-Up Table (PCS LUT). 
**Problem:** Symmetrical pitch class sets (Tritones, Augmented Triads, and Diminished 7th chords) are inherently ambiguous and cannot be correctly spelled using a static, single-interpretation lookup. They require dynamic, context-aware spelling based on the current musical key and the lowest played note (inversion).
**Objective:** Implement an Isolated Context Interception Algorithm that intercepts the standard PCS LUT spelling for symmetrical chords and dynamically overrides it with the correct enharmonic rotation array, preserving theoretical accuracy on the notation canvas.

## 2. Technical Decisions & Dependencies
* **Architecture:** We will implement a pure-function interceptor pattern in a new utility file (`src/utils/symmetricalSpeller.ts`) that is called by the existing `src/utils/chordSpeller.ts`. 
* **Math (Distance Normalization):** The core logic relies on normalizing the lowest note's pitch class distance from the global key center. The formula is:
    `const distance = (lowNotePitchClass - keyCenterPitchClass + 12) % 12;`
* **Data Structures:** * Instead of complex `if/else` chains, you will utilize O(1) `Map` or plain object dictionaries associating the calculated `distance` integer (`0-11`) to the required degree rotation array.
    * Tritone Intervals: `[0, 6]` (Length: 2)
    * Augmented Triad Intervals: `[0, 4, 8]` (Length: 3)
    * Diminished 7th Intervals: `[0, 3, 6, 9]` (Length: 4)
* **Dependencies:** No new external dependencies required. Utilize existing TypeScript/Vite setup and Vitest for TDD.

### Phase 1: Pure Mathematical Logic & Lookup Maps — ✅ Completed
* **Objective:** Implement the distance calculator and lookup maps for Tritones, Diminished 7ths, and Augmented Triads based on the provided logic.

        * **Tritone:** * Distances `[1, 4, 6, 8, 9, 11]` -> `["1", "b5"]`
