# Product Requirements Prompt (PRP) — 🔘 Pending

## 1. Project Context & Objectives
**Critical Bug Fix Operation:** The current selection state is severely broken due to React "Stale Closures" inside event listeners, missing array interpolation for Shift-clicks, and faulty Marquee intersection math.
* Standard clicks and Cmd-clicks are failing or acting inconsistently because the `onClick` handlers are referencing a stale `selectedPitches` state.
* Shift-click is functioning as an additive (Cmd) click instead of a range selector.
* Marquee selection clears the board instead of selecting intersected elements.

## 2. Technical Decisions & Dependencies
* **Functional State Updates Mandate:** You are strictly forbidden from reading the `selectedPitches` state directly inside `onClick`, `onMouseMove`, or `onMouseUp` handlers to determine the next state. You MUST use the functional updater pattern: `setSelectedPitches(prevSelected => { ...logic... return nextSelected; })`.
* **Shift-Click Algorithm:** To select a range, you must extract all active pitches, sort them, find the indices of the `lastSelectedPitchRef.current` and the newly clicked pitch, slice the array between those minimum and maximum indices, and add them to the set.
* **Marquee Coordinate Mapping:** The marquee intersection is failing because the `onMouseUp` is calculating rects incorrectly or wiping the state. The marquee MUST also use the functional state updater to merge or replace the selection.

## 3. Implementation Phases

### Phase 1: Eradicate Stale Closures in Click Handlers — 🔘 Pending
* **Objective:** Guarantee that click and Cmd-click modifiers always read the most up-to-date selection state.
* **Instructions:**
    1. In `NotationCanvas.tsx`, locate the `onClick` handler attached to the notehead/accidental.
    2. Rewrite the state mutation to strictly use the functional updater pattern:
       ```typescript
       setSelectedPitches(prev => {
           const next = new Set(prev);
           // ... modifier logic here ...
           return next;
       });
       ```
    3. **Standard Click:** If no modifiers are pressed, return `new Set([clickedPitch])`.
    4. **Cmd/Ctrl Click:** If `e.metaKey || e.ctrlKey`, check `if (prev.has(clickedPitch))` -> `next.delete(clickedPitch)`, else `next.add(clickedPitch)`. Return `next`.
    5. Ensure `lastSelectedPitchRef.current = clickedPitch;` is updated on every click.
    6. Ensure the canvas background click handler also uses `setSelectedPitches(() => new Set())`.

### Phase 2: Shift-Click Range Interpolation — 🔘 Pending
* **Objective:** Implement the math to select all notes between two clicked points in a sonority.
* **Instructions:**
    1. Inside the functional updater from Phase 1, add the `e.shiftKey` condition.
    2. If `e.shiftKey` is true, check if `lastSelectedPitchRef.current` exists. If not, just select the clicked pitch.
    3. If it does exist, extract all active pitches: `const sortedPitches = Array.from(activeNotes.current.keys()).sort((a, b) => a - b);`
    4. Find `idxA = sortedPitches.indexOf(lastSelectedPitchRef.current)` and `idxB = sortedPitches.indexOf(clickedPitch)`.
    5. Slice the range: `const range = sortedPitches.slice(Math.min(idxA, idxB), Math.max(idxA, idxB) + 1);`
    6. Return a new Set containing these pitches. (If `Cmd` is NOT held with `Shift`, replace the set; if `Cmd` IS held, merge the range into `prev`).
* **TDD Checkpoint:**
    * **Test Case 1:** Given `activeNotes` `[60, 64, 67, 72]`. Click `60`. Shift-click `67`. Assert `selectedPitches` equals `[60, 64, 67]`.
    * **AGENT INSTRUCTION:** Set Phase status to 🔵 Active. Implement range interpolation. Once tests PASS, set status to ✅ Completed.

### Phase 3: Marquee Stale State & Math Fix — 🔘 Pending
* **Objective:** Ensure marquee box accurately populates the selection without wiping it due to stale state.
* **Instructions:**
    1. In the `onMouseUp` handler for the marquee, capture the intersecting pitches into a temporary array: `const intersectedPitches: number[] = [];`
    2. Run the `getBoundingClientRect()` intersection math as before. For every intersecting note, push its `data-midi-note` to `intersectedPitches`.
    3. Update the state using the functional updater:
       ```typescript

           if (intersectedPitches.length === 0) return new Set(); // Or return prev if we want marquee clicking empty space to do nothing

           return next;
       });
       ```
