## 1. Project Context & Objectives
The spelling engine is currently interpreting isolated Major 2nd sonorities (Pitch Class Set decimal `5`, i.e., root and major second `{0, 2}`) using sharp-side enharmonics when out of key context (e.g., spelling MIDI notes `[58, 60]` as `[A#, B#]` instead of the standard `[Bb, C]`). 

To resolve this, we must update the internal heuristic that governs chromatic root spelling for cardinality-2 sets so that decimal `5` defaults to the flat-side index of the chromatic interval array.

## 2. Technical Decisions & Dependencies
* **Enharmonic Routing Array:** The function `getChromaticRootInterval` within `src/utils/chordSpeller.ts` uses an array named `check14679e` to route specific 2-note sets to the flat-side interval (`chromaticIntervals[rootPCN][0]`). 
* **Mathematical Solution:** A Major 2nd interval corresponds to the pitch class set `{0, 2}`. Using the summation formula $\sum 2^{pc}$, this results in $2^0 + 2^2 = 1 + 4 = 5$. By injecting `5` into the `check14679e` lookup array, the engine will deterministically route Major 2nds to the flat-side chromatic array.
* **No Side Effects:** This is a strictly confined logical update to a pure function; no changes to `notationMath.ts` or component rendering layers are necessary.
