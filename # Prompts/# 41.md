# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
Chords built on Fb (e.g., the bII chord in Eb Major, MIDI notes `[64, 68, 71]`) are failing to spell correctly in the individual note notation, despite the chord symbol generating the correct name ("Fb"). 

The root cause is a missing key in the `PITCH_TO_PC` dictionary in `src/utils/chordSpeller.ts`. Because `"Fb"` is absent, the fallback logic evaluates it as `"F"` (Pitch Class 5 instead of 4). When calculating the relative interval between the key (Eb, PC 3) and the root (Fb), it erroneously calculates a Major 2nd (Eb -> F) instead of a minor 2nd (Eb -> Fb). This upstream error cascades through the interval summation engine, stripping the flats from the resulting spelled triad (yielding F, A, C instead of Fb, Ab, Cb).

## 2. Technical Decisions & Dependencies
* **Dictionary Amendment:** We must explicitly add `"Fb": 4` to the `PITCH_TO_PC` export. This is the only missing standard accidental in the dictionary (Cb, E#, and B# are already present).
* **No Side Effects:** This is a pure data dictionary fix and will immediately resolve the mathematical downstream interval calculations.
