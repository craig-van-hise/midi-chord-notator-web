# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
**Objective:** Hard-wire the audio engine to respect Toggle Mode state resolutions to prevent infinite sustain, and expand the 500ms preview playback to keyboard navigation and transposition events.
**Current State:** Toggling an active note "OFF" via the virtual keyboard sends a `0x90` message that triggers an orphaned `audioEngine.noteOn`, resulting in infinite sustain because the note is deleted from state before a release can be issued. Keyboard-driven selection changes and transpositions are currently silent.
**End State:** The MIDI listener will evaluate the *result* of a Toggle action before firing audio. If `0x90` results in a note removal, it will explicitly fire `audioEngine.releaseNote`. Transposition and arrow-key selection traversal will trigger the 500ms preview helper.

## 2. Technical Decisions & Dependencies
* **Toggle State Resolution:** The worker agent MUST place the `audioEngine` triggers *inside* the state evaluation branches of `isNoteOn` in `NotationCanvas.tsx`, not at the top of the event handler.
* **Preview Batching:** When multiple notes are transposed simultaneously via keyboard shortcuts, the agent MUST loop over the mutated selection array and trigger `playPreviewNote` for all selected pitches simultaneously.
