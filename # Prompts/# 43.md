# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
The spelling engine is currently defaulting to sharp-side enharmonics for isolated minor 7th sonorities (Pitch Class Set decimal `1025`, i.e., `{0, 10}`). Similar to the minor 3rd exception implemented previously, when a minor 7th is built on the `b3` or `b7` degrees relative to the current key, it should be spelled using flat-side enharmonics. 

For example, in C Major, playing MIDI notes `[63, 73]` (built on the `b3` degree, `rootPCN` 3) currently defaults to sharp spellings. It should be spelled as `[Eb, Db]`. Similarly, `[70, 80]` (built on the `b7` degree, `rootPCN` 10) should be spelled as `[Bb, Ab]`. This PRP groups this new mathematical condition with the existing minor 3rd exception.

## 2. Technical Decisions & Dependencies
* **Exception Injection:** We will modify the explicit `if` condition injected in the previous phase within `getChromaticRootInterval`. We need to expand the condition to intercept either `decimal === 9` (m3) OR `decimal === 1025` (m7) when the relative pitch class number (`rootPCN`) is either `3` or `10`.
* **Mathematical Solution:** A minor 7th corresponds to the pitch class set `{0, 10}`, yielding $\sum 2^{pc} = 2^0 + 2^{10} = 1 + 1024 = 1025$. By grouping `9` and `1025` together, we route both to `chromaticIntervals[rootPCN][0]` (the flat-side index) under these specific root conditions.
* **No Side Effects:** This remains a strictly confined logical update to a pure function.
