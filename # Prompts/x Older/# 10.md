# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
This phase introduces essential professional features to the `MIDI Chord Notator`. We are refining the UI layout (Header height, Notation width), implementing robust MIDI device routing constraints (preventing loopbacks), and adding new interactive layers. Crucially, the virtual keyboard must become an input device, supported by a fully functional Sustain Pedal (CC 64) state queue to allow users to build chords interactively. A global MIDI Panic function and an Information modal are also required.

## 2. Technical Decisions & Dependencies
* **Files to Modify:** `src/App.tsx`, `src/midi/MidiPortSelector.tsx`, `src/midi/MIDIProvider.tsx`, `src/components/NotationCanvas.tsx`, `src/components/Keyboard.tsx`.
* **Keyboard Interactivity:** Use `onPointerDown`, `onPointerUp`, and `onPointerLeave` rather than mouse events to guarantee touch/mobile compatibility.
* **Sustain Queue Algorithm:** To prevent stuck notes, `MIDIProvider` must act as an event interceptor. It will track `physicallyHeldNotes` (Set) and `pendingNoteOffs` (Set).
* **MIDI Panic:** An explicit loop must fire `Note Off` messages for all 128 notes across all 16 channels to all connected outputs, followed by `CC 123` (All Notes Off) and `CC 64` (Sustain = 0).

* **Objective:** Allow the virtual keyboard to act as a MIDI generator and implement CC 64 state management.
* **Instructions:**
    1. **Target `src/components/Keyboard.tsx`:**
       * Add a "Sustain Pedal" toggle UI (a button or stylized checkbox) near the keyboard.
       * Wire individual keys to trigger `dispatchVirtualMidi(note, isNoteOn)`. 
       * Use `onPointerDown` (Note On, velocity 100), `onPointerUp` (Note Off), and `onPointerLeave` (Note Off) to simulate physical key presses.
       * These virtual events must route back to `MIDIProvider` as if they came from hardware.
    2. **Target `src/midi/MIDIProvider.tsx`:**
       * Add states: `isSustainActive` (boolean, mapped to the UI toggle AND incoming CC 64), `physicallyHeldNotes` (Set), `pendingNoteOffs` (Set).
       * **Intercept Logic:** When handling an incoming MIDI message (or virtual message):
         * **CC 64:** If data is `[0xBn, 64, val]`. Update UI toggle. If `val < 64` (turning off): iterate `pendingNoteOffs`, dispatch Note Off events to the rest of the app, and clear the set.
         * **Note On:** Add to `physicallyHeldNotes`. Dispatch event to app. Remove from `pendingNoteOffs` if present.
         * **Note Off:** Remove from `physicallyHeldNotes`. If `isSustainActive` is true, add to `pendingNoteOffs` and **DO NOT** dispatch the Note Off event. If false, dispatch the Note Off event.
* **TDD Checkpoint:**

    * **Test Case 2:** [Given `isSustainActive = true`, When Note Off 60 arrives, Assert Note Off event is NOT dispatched to canvas, but queued in `pendingNoteOffs`]
