## 1. Project Context & Objectives
The "Glass Panel" event capture system successfully restored click selection and transpositions. However, two issues remain: the visual Marquee selection box is not rendering, and transposing a note into a unison with an existing note causes the note to be erased (Unison Erasure). The objective is to imperatively render the visual Marquee box below the Glass Panel and refactor the selection/transposition pipeline to support duplicate pitches using instance tracking.

## 2. Technical Decisions & Dependencies
* **Imperative Marquee Rendering:** To prevent React render lag during drag, the visual marquee must be a dedicated `<div>` controlled purely via a React `useRef` and inline style mutations (`marqueeRef.current.style.width = ...`). It must sit below the Glass Panel (`z-40`).
* **Unison Instance Tracking:** `activeNotes` must be allowed to contain duplicate integers (e.g., `[60, 60, 67]`). 
* **Selection ID Refactoring:** The `selectedNoteIds` Ref MUST NOT store raw MIDI pitches. It must store a composite ID based on the pitch and its original index (e.g., `"60-0"`, `"64-1"`) or a generated UUID. When a note is transposed, its specific index/instance must be updated without erasing duplicates.
