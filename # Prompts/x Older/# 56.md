# Product Requirements Prompt (PRP) — 🔘 Pending

## 1. Project Context & Objectives
The bottom edge of the white keys on the virtual keyboard is currently clipping, resulting in a flat, unnatural appearance that cuts off the `border-radius`. 

The root cause is a CSS Box Model conflict. The parent `piano-container` has a strict height of `88px` (`WHITE_KEY_HEIGHT`) and an `overflow: hidden` mask. However, because the parent also has a `1px` top border and global Tailwind rules apply `box-sizing: border-box`, the internal rendering area is actually constrained to `87px`. This forces the `88px` tall white keys to overflow by 1px, causing their bottom border-radius to be literally chopped off by the parent's hidden overflow mask. Additionally, the white keys lack an explicit `borderBottom` definition, causing them to bleed into the background.

We will resolve this by setting the parent container to dynamically wrap the keys, aligning the border-radius clipping masks, and explicitly drawing the bottom edges.

## 2. Technical Decisions & Dependencies
* **Dynamic Height:** Changing the parent container's height from a hardcoded `88px` to `fit-content` allows it to wrap the fixed-height children perfectly without sub-pixel clipping.
* **Clipping Mask Alignment:** The white keys have a bottom radius of `3px`. The parent container has a uniform radius of `4px`. We will update the parent to `4px 4px 3px 3px` so the container's overflow mask aligns perfectly with the children's outer corners.
* **Edge Definition:** Injecting `borderBottom: '1px solid #ccc'` into both the parent container and the individual white keys will restore the defined "lip" of physical piano keys.
