# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
The application is currently suffering from a "Crossover Desync Bug." Because note selection is tracked via the sorted array index (e.g., `Index 1 is selected`), transposing a note past another note causes the array to re-sort, which transfers the selection to the wrong note. Additionally, removed notes leave "ghost" selections. The objective is to replace array-index tracking with Immutable Unique IDs (UUIDs) for every note instance, ensuring a specific voice remains selected regardless of its sorted position.

## 2. Technical Decisions & Dependencies
* **Immutable IDs:** You must update the global note state interface (e.g., `ActiveNoteData` or whatever `activeNotes` contains) to include an `id: string` property.
* **ID Generation:** Use `crypto.randomUUID()` (or `Date.now().toString() + Math.random()` if crypto is unavailable) to assign a permanent ID to every note the exact moment it is added to the active state.
* **Global Sync:** The selection state (`selectedNoteIds`) must store these UUIDs, NOT raw MIDI numbers and NOT array indices.
