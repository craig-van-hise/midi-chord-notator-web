## 1. Project Context & Objectives
The previous UI phase introduced critical layout and state bugs. The header `<h1>` is improperly sized and lacks flex constraints, causing the header to overflow and clip content on the right. The root application wrapper has arbitrary horizontal margins/padding that waste screen real estate. The MIDI port selectors fail to accept `"none"` values due to poor state handling, and the loopback prevention logic continues to fail. 

Additionally, the `KeySignatureSelector` must be relocated from the header to the `NotationCanvas` to sit adjacent to the Grand Staff bracket.
