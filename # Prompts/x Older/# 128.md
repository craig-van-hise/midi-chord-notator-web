# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
During a previous architectural refactor to support hardware MIDI sustain envelopes, the default `APP_TRANSFORM` audio preview was silenced to prevent double-triggering. However, we failed to explicitly restore the 500ms auto-preview for mouse clicks on the UI arrows. We need to pass an `isUiClick` flag through the Event Bridge to selectively trigger the short audio preview only when the user clicks the screen.

## 2. Implementation Phases

### Phase 1: Selective UI Audio Preview — ✅ Completed
* **Objective:** Pass an `isUiClick` flag to `APP_TRANSFORM` and conditionally execute `playPreviewNotes` in the Canvas.
* **Instructions:**
  1. Open `src/components/toolbar/TransformationsDrawer.tsx` (or `TransformationsContainer.tsx`, wherever `handleButtonDown` lives).
  2. Locate the `handleButtonDown` function. Update the `APP_TRANSFORM` dispatch to inject `isUiClick: !!e` into the detail payload:
     ```typescript
     window.dispatchEvent(new CustomEvent('APP_TRANSFORM', { 
       detail: { 
         type: id, 
         stepSize: configs[id].stepSize, 
         isUiClick: !!e 
       } 
     }));
     ```
  3. Open `src/components/NotationCanvas.tsx`.
  4. Locate the `APP_TRANSFORM` event listener.
  5. At the very bottom of the `APP_TRANSFORM` handler block (after the state updates and layout recalculations), append this conditional check:
     ```typescript
     if ((e as CustomEvent).detail.isUiClick) {
         const transposedStrings = Array.from(selectedNoteIds.current)
             .map(id => activeNotes.current.find(n => n.id === id)?.note)
             .filter((n): n is number => typeof n === 'number')
             .map(pitch => Tone.Frequency(pitch, "midi").toNote());
         
         if (transposedStrings.length > 0) {
             playPreviewNotes(transposedStrings, true);
         }
     }
     ```
