## 1. Primary Regression
**File:** `src/utils/chordSpeller.ts`
**Nature:** Logic Deletion / Safety Regresson
**Commit Reference:** `202789e` (Refactor to disambiguate "root")

### Mutation Detail:
The safety guard checking for the existence of a LUT entry was removed. This causes an immediate `TypeError` if a pitch set is not found in the database (or if the database is not yet loaded).

## 3. Impact Analysis
The removal of the safety check in `getRootSpellingFromKey` combined with the new MIDI integration means that any transient state where the LUT is empty or a note is invalid will now crash the entire UI thread instead of failing gracefully with a "C" root spelling.
