# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
The previous phase successfully introduced the `splitPoint` logic for bifurcating Notehead Y-coordinates across the physical visual gap between the Treble and Bass staves. However, the implementation failed on two fronts:
1. **UI Failure:** The Settings Modal was implemented as a localized popover with hardcoded, insufficient options, and styling defects.
2. **Mathematical Decoupling (Ledger Lines):** The ledger line generation logic was not updated to inherit the `isTrebleGroup` state. Consequently, ledger lines are still calculating their Y-positions from absolute center, causing them to detach from their parent noteheads and render on the wrong clef entirely (e.g., C4 notehead renders correctly below Treble, but its ledger line renders above Bass).

## 2. Technical Decisions & Dependencies
* **Files to Modify:** `src/components/SettingsModal.tsx` (or equivalent), `src/components/NotationCanvas.tsx`.
* **Modal Architecture:** Replace the popover with a standard full-screen, semi-transparent backdrop and a centered dialog box.
* **Split Point Options Range:** Generate an exhaustive `<select>` list from C3 (MIDI 48) to C5 (MIDI 72). 
* **Ledger Line Inheritance Mandate:** Ledger line Y-coordinates MUST inherit the exact `isTrebleGroup` offset applied to the notehead. The simplest and most deterministic way to achieve this is to inject ledger lines as absolutely positioned children *inside* the `noteContainer`, rendering them relative to the notehead itself (e.g., `top: 0` for the note's center line) rather than recalculating screen coordinates.

### Phase 2: Ledger Line Anchoring (The Inheritance Fix) — ✅ Completed
* **Objective:** Force ledger lines to strictly follow the notehead by injecting them inside the already-positioned `noteContainer`.
* **Instructions (Target: `src/components/NotationCanvas.tsx` -> `isNoteOn` block):**
    1. Locate the block where `noteContainer`'s `top` style is calculated (which correctly applies the `isTrebleGroup` visual gap offset).
    2. Completely delete the existing, separate ledger line DOM injection logic.
    3. Implement a new ledger line generator *inside* the note creation logic:
       * **Middle C/Gap Rules:** If `stepOffset === 0`, create a ledger line `<div>`. Set its style to `position: absolute; top: 50%; left: 50%; transform: translate(-50%, -50%);`. Append it directly to `noteContainer`. By living inside `noteContainer`, it is mathematically guaranteed to perfectly bisect the notehead, regardless of the `splitPoint`.
       * **Out-of-Bounds Rules (Above Treble / Below Bass):** If the note requires multiple ledger lines (e.g., `stepOffset >= 12`), calculate the number of lines required. Loop to generate them. Append them to `noteContainer`. 
       * *Math Directives for Multiple Lines:* Since they are inside `noteContainer`, the line passing through the note is at `top: 50%`. The line below the note is at `top: calc(50% + var(--staff-space))`. The line above the note is at `top: calc(50% - var(--staff-space))`.
* **TDD Checkpoint:**
