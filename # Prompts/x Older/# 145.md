# Product Requirements Prompt (PRP) — Upstream Filter Remediation 🔘

## 1. Project Context & Objectives
**Project:** Grand Staff MIDI Notator
**Current State:** The `applyNoteFilter` logic was incorrectly injected at the very end of the signal chain (just before `audioEngine.triggerAttack`). This protected the audio but allowed out-of-bounds notes to crash the visual notation state and main UI keyboard.
**Objective:** Architecturally relocate the filter upstream. The mathematical transformation engine must pipe its output *directly* through the filter before updating ANY React visual state or triggering audio. This guarantees the entire application shares a single, mathematically safe source of truth.

## 2. Technical Decisions & Dependencies
* **Signal Flow Correction:** * *OLD FLOW:* Math -> `setVisualState(badNotes)` -> `applyFilter(badNotes)` -> `audioEngine(goodNotes)`.
  * *NEW FLOW:* Math -> `applyFilter(calculatedNotes)` -> `const safeNotes` -> `setVisualState(safeNotes)` & `audioEngine(safeNotes)`.
* **State Unification:** Because the filter now sits upstream, the main UI Keyboard, the SMuFL notation canvas, and the audio engine will inherently sync. An "octave wrap" transformation will physically appear wrapped on the main keyboard.

## 3. Implementation Phases

### Phase 1: Upstream Injection & Downstream Cleanup — 🔘 Pending
* **Objective:** Remove the downstream audio filter and apply it immediately to the transformation output.
* **Instructions:**
  1. Open `src/midi/MIDIProvider.tsx` (and any file containing transformation handlers like `applyChromaticShift`, `applyPcsRotation`, etc.).
  2. Locate the calculation of the transformed notes (e.g., `const transformedNotes = ...`).
  3. Immediately pipe this result through the filter:
     ```typescript
     const safeFilteredNotes = applyNoteFilter(transformedNotes, filterMode, filterRange);
     ```
  4. Replace all downstream usages of `transformedNotes` with `safeFilteredNotes`.
     * The React state setter (e.g., `setNotationState` or `setActiveNotes`) MUST receive `safeFilteredNotes`.
     * The mapping reference `activeTransformationNotesRef.current.set(trigger, safeFilteredNotes)` MUST receive `safeFilteredNotes`.
  5. Remove the redundant `applyNoteFilter` call that was previously placed right before `audioEngine.triggerAttack()`. The audio engine should now just blindly play the `safeFilteredNotes` array.
