## 1. Project Context & Objectives
The spelling engine is currently utilizing sharp-side enharmonics for the root note of isolated Perfect 4th sonorities (Pitch Class Set decimal `33`, i.e., `{0, 5}`) when the root falls outside the active diatonic key. For example, playing MIDI notes `[61, 66]` in C Major defaults to `[C#, F#]`. 

To resolve this, we must update the chromatic root spelling heuristic for cardinality-2 sets so that decimal `33` defaults to the flat-side interval array, returning `[Db, Gb]`.

## 2. Technical Decisions & Dependencies
* **Enharmonic Routing Array:** The `getChromaticRootInterval` function uses the `check14679e` array to route specific 2-note pitch class sets to the flat-side chromatic interval (`chromaticIntervals[rootPCN][0]`). 
* **Mathematical Solution:** A Perfect 4th interval corresponds to the pitch class set `{0, 5}`. Using the standard summation formula $\sum 2^{pc}$, this is $2^0 + 2^5 = 1 + 32 = 33$. Injecting `33` into the `check14679e` lookup array will deterministically route the root of Perfect 4th sonorities to the flat side when falling on a non-diatonic pitch class.
* **No Side Effects:** This remains a strictly confined logical update to a pure function; no changes to the visual layout engine are necessary.
