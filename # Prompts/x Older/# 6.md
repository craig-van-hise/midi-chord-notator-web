# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
The application now correctly handles notehead collisions using the zipper pattern. However, as illustrated by your feedback, a secondary collision domain exists: accidentals. Currently, accidentals are statically anchored to their parent noteheads at `-1.5 * staffSpace`. When notes are stacked closely (a 5th or less), their accidentals overlap vertically. Furthermore, if a notehead is shifted right by the zipper pattern, its accidental is dragged into the collision zone of the unshifted notes below it.

We must implement a deterministic "Greedy Column Assignment" algorithm to dynamically calculate the X-offset of every accidental in the active chord, mirroring professional engraving rules (e.g., Dorico's leftward staggering).

## 2. Technical Decisions & Dependencies
* **Files to Modify:** `src/components/NotationCanvas.tsx`.
* **Collision Threshold:** Accidentals collide if their vertical diatonic distance is `<= 3` steps. `Math.abs(noteA.stepOffset - noteB.stepOffset) <= 3`.
* **Algorithm (Greedy Column Placement):**
    1. Filter active notes in the staff group to only those with accidentals.
    2. Sort descending by pitch (highest `stepOffset` first).
    3. Attempt to place each accidental in `Column 0`. If it collides with an existing accidental in that column, try `Column 1`, etc., until an empty slot is found.
* **Math Overrides (The Zipper Compensation):** If a notehead was pushed right (`isShiftedRight`) during the zipper phase, its accidental must be pushed left by an equal and opposite amount to maintain the correct baseline before the column staggering is even calculated.

## 3. Implementation Phases

### Phase 1: The Accidental Staggering Algorithm — ✅ Completed
* **Objective:** Implement the logical grouping engine to assign a column index to each accidental.
* **Instructions (Target: `src/components/NotationCanvas.tsx`):**
    1. Beneath the `applyNoteheadOffsets` function created in the previous WO, create `applyAccidentalOffsets(staffGroup, shiftedNoteIds)`. 
       *Note: `shiftedNoteIds` is a `Set` or `Array` of note IDs that were shifted right during the zipper algorithm. You must update `applyNoteheadOffsets` to return this list.*
    2. Filter `staffGroup` values to an array of notes where `note.accidental != null`.
    3. Sort the array descending: `notes.sort((a, b) => b.stepOffset - a.stepOffset)`.
    4. Implement the Greedy Column array: `const columns: number[][] = [];`
    5. Iterate through the sorted notes:
       ```javascript
       let assignedColumn = 0;
       let placed = false;
       while (!placed) {
           if (!columns[assignedColumn]) columns[assignedColumn] = [];
           // Check for collisions in this column
           const collision = columns[assignedColumn].some(existingOffset => 
               Math.abs(existingOffset - note.stepOffset) <= 3
           );
           if (!collision) {
               columns[assignedColumn].push(note.stepOffset);
               note.accidentalColumn = assignedColumn; // Store it for Phase 2
               placed = true;
           } else {
               assignedColumn++;
           }
       }
       ```

### Phase 2: Imperative DOM Manipulation for Accidentals — ✅ Completed
* **Objective:** Apply the calculated column indices as CSS horizontal translations, compensating for zipper-shifted parent containers.
* **Instructions (Target: `src/components/NotationCanvas.tsx`):**
    1. Define the spacing constants:
       `const ACC_BASE_OFFSET = -1.5;` (multiplier of staffSpace)
       `const ACC_COLUMN_WIDTH = 1.2;` (multiplier of staffSpace)
       `const NOTE_SHIFT_COMPENSATION = 1.2;` (multiplier of staffSpace)
    2. Inside `applyAccidentalOffsets`, after assigning columns, iterate through the notes again to update the DOM.
    3. Locate the accidental DOM element inside the parent `noteContainer`.
    4. Calculate the base X offset: 
       `let offsetMultiplier = ACC_BASE_OFFSET - (note.accidentalColumn * ACC_COLUMN_WIDTH);`
    5. Apply Zipper Compensation: If `shiftedNoteIds.has(note.id)`, subtract the compensation so the accidental is pushed back to the main chord baseline:
       `if (isShifted) offsetMultiplier -= NOTE_SHIFT_COMPENSATION;`
    6. Apply the style: `accidentalEl.style.left = calc(${offsetMultiplier} * var(--staff-space))`.
    7. Ensure `applyAccidentalOffsets` is called immediately after `applyNoteheadOffsets` whenever a MIDI Note On/Off event fires.
