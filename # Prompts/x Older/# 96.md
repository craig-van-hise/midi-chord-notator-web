# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
This sprint addresses three critical bugs in the global state synchronization and history engines. 

**The Bugs:**
1. **Maximum Call Stack Size Exceeded:** `NotationCanvas` is triggering an infinite loop by broadcasting an `updateActiveNotes` payload in response to receiving a `refresh` event. 
2. **Missing Undo Hook:** Hardware and virtual keyboard toggles are bypassing the Undo/Redo stack because `commitState()` is missing from the simultaneous-mode logic branches.
3. **Visual Desync / Flickering:** The `Keyboard` component is prematurely erasing its visual highlights and "Spelled Notes Strip" based on raw hardware `Note Off` events, entering a race condition with the computational `refresh` payloads that dictate the true global state.

## 2. Technical Decisions & Dependencies
* **Break the Loop:** The `refresh` block inside `NotationCanvas`'s `handleMidiMessage` must act exclusively as a receiver. It must never call `updateActiveNotes`.
* **State Source of Truth:** `Keyboard.tsx` must abandon its raw hardware `isNoteOn` / `isNoteOff` logic. It will derive its visual state (key highlights and note strip) *entirely* from the `refresh` payload broadcasted by `NotationCanvas`.
