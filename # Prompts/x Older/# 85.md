## 1. Project Context & Objectives
An objective code audit revealed two critical logic flaws in `NotationCanvas.tsx`:
1. **Spelling Desync (Stave Inversion):** Transposing notes causes the `activeNotes` array to become unsorted. The `getChordSpelling` function returns spellings based on sorted order, resulting in `updateSpellings` applying the wrong Y-coordinate (`stepOffset`) to the wrong note. 
2. **React DOM Ghosting:** The rendering loop uses `note.note` (pitch) as the React `key`. During crossover transpositions, React reuses the wrong DOM elements, creating visual phantom notes due to the 75ms CSS transition.

## 2. Technical Decisions & Dependencies
* **Pre-Spelling Sort:** The `activeNotes.current` array MUST be sorted low-to-high by `note.note` immediately inside `updateSpellings()` before it is passed to `getChordSpelling`.
* **Immutable DOM Keys:** The JSX rendering loop MUST use `note.id` for the `key` attribute to guarantee React flawlessly tracks the DOM nodes regardless of array position or pitch.
