# Product Requirements Prompt (PRP) — 🔘 Pending

## 1. Project Context & Objectives

You are tasked with evolving the Grand Staff MIDI Notator from a passive real-time visualizer into an interactive notation editor. You will implement standard selection patterns (Single, Multi/Cmd, Range/Shift, Marquee Drag) and a suite of keyboard-driven editing shortcuts (chromatic, diatonic, octave transpositions, and chord inversions).

Because the rendering engine uses imperative DOM manipulation (`activeNotes.current`, `recalculateLayout()`) to maintain low-latency performance, **you must deeply integrate these interaction states without causing React reconciliation lag or breaking the MIDI Thru data loop.**

## 2. Technical Decisions & Dependencies

To eliminate architectural drift, strictly adhere to the following decisions:

* **Selection State:** Implement `selectedPitches = React.useState<Set<number>>(new Set())`. Selected notes must be visually differentiated via CSS (e.g., `filter: drop-shadow(0 0 8px #aa3bff)` or a high-contrast color class) applied during the mapping in `renderedNotes`.
* **Hit Testing (Marquee):** Implement custom DOM Rect intersection logic. Do NOT install heavy drag-and-drop libraries. Attach standard `onMouseDown`, `onMouseMove`, `onMouseUp` handlers to `.notation-canvas-container`. Use the existing `data-midi-note` attribute to tie DOM elements back to their logical pitch.
* **Detached Execution:** Modifying a note (e.g., transposing C4 to D4) while the physical MIDI key is held down causes a state fracture. **Rule:** When a note is transposed, immediately delete its original pitch from `physicalKeysDown.current` to prevent ghost note-offs from clearing the newly edited pitch.
* **Diatonic Transposition Algorithm ($O(1)$ lookup):** You must use the existing `getDiatonicMap(keySignature)` from `notationMath.ts`.
* Extract the keys from the Map to get the 7 active scale pitch classes.
* To transpose a pitch: find its base pitch class (`pitch % 12`). Find the nearest scale pitch class. Advance/retreat by 1 index in the scale array. Add back the octave offset.

### Phase 1: Interactive Selection Foundation — 🔘 Pending

* **Objective:** Enable clicking individual notes to select them, supporting single, additive (Cmd/Ctrl), and range (Shift) selections.
* **Instructions:**
1. In `NotationCanvas.tsx`, define state: `const [selectedPitches, setSelectedPitches] = useState<Set<number>>(new Set());`
2. Add a `lastSelectedPitch` ref to track the anchor for Shift-clicking.
3. Add an `onClick` handler to the note `div` (the one with `data-midi-note`).
4. **Click Logic:**
* Standard click: Clear set, add pitch.
* `Cmd/Ctrl` + click: Toggle pitch in set.
* `Shift` + click: Find all pitches in `activeNotes.current` between `lastSelectedPitch` and the current clicked pitch (inclusive, sorted mathematically), and add them to the set.


5. Update the inline styles or Tailwind classes of the notehead text node to conditionally glow or change color if `selectedPitches.has(note.note)`.

### Phase 3: Transposition Mathematics Engine — 🔘 Pending

* **Objective:** Build pure functions for musically accurate transpositions.
* **Instructions:**
1. Create `src/utils/editingMath.ts`.
2. Implement `transposePitch(pitch: number, amount: number): number` (Clamped between 0-127).
3. Implement `transposeDiatonic(pitch: number, direction: 1 | -1, keySignature: string): number`.
* *Logic:* `const scaleMap = getDiatonicMap(keySignature); const scalePCs = Array.from(scaleMap.keys()).sort((a,b)=>a-b);`
* Determine the current pitch class (`pc = pitch % 12`) and octave base. Find the index of `pc` in `scalePCs` (or the closest index if accidental). Add `direction` to the index, handling array wrapping and octave incrementing/decrementing.


4. Implement `invertChord(pitches: number[], direction: 1 | -1): number[]`.
* *Up (1):* Shift the lowest pitch up by 12 until it is strictly greater than the current highest pitch.
* *Down (-1):* Shift the highest pitch down by 12 until it is strictly less than the current lowest pitch.




* **TDD Checkpoint:** * **Test Case 1:** `transposeDiatonic(60 /* C4 */, 1, 'C Major')` -> Expect `62 /* D4 */`.
* **Test Case 2:** `transposeDiatonic(64 /* E4 */, 1, 'C Major')` -> Expect `65 /* F4 */`.
* **Test Case 3:** `invertChord([60, 64, 67], 1)` -> Expect `[64, 67, 72]`.
* **AGENT INSTRUCTION:** Set Phase status to 🔵 Active. Write tests in `editingMath.test.ts`. Once they PASS, set status to ✅ Completed.



### Phase 4: Keyboard Navigation & Execution — 🔘 Pending

* **Objective:** Wire keyboard shortcuts to mutate the canvas state.
* **Instructions:**
1. In `NotationCanvas.tsx`, attach a `useEffect` listening to `keydown` on the `window` (ensure you check that `document.activeElement` is not an input field).
2. Intercept Arrow Keys: Up/Down/Left/Right. Prevent default scrolling behavior.
3. **Left/Right Navigation:** As requested, wire as placeholder. Simply cycle the `selectedPitches` to the next/prev pitch in `activeNotes` numerically.
4. **Transposition Triggers:** Target only notes currently in `selectedPitches`. (If none selected, optionally target all `activeNotes`).
* `Shift + Up/Down` -> `transposePitch(+1 / -1)`
* `Option/Alt + Up/Down` -> `transposeDiatonic(direction, keySignature)`
* `Cmd/Ctrl + Up/Down` -> `transposePitch(+12 / -12)`
* `Option/Alt + Cmd/Ctrl + Up/Down` -> `invertChord`


5. **Execution Loop:** * Calculate the new set of pitches.
* Remove old pitches from `activeNotes.current` AND `physicalKeysDown.current`.
* Insert new pitches into `activeNotes.current`.
* Update `selectedPitches` to match the newly transposed values.
* Imperatively call `updateSpellings()` and `recalculateLayout()` to force the render cycle.
