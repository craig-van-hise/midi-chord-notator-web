## 2. Technical Decisions & Dependencies
* **Render Loop Guarantee:** `recalculateLayout()` MUST be executed in a `finally` block within `updateSpellings()`. The canvas must always render the physical notes currently held in memory, even if theoretical chord identification fails.
* **Hit-Test Precision:** The Pythagorean distance threshold in `getNoteAtCoordinates` must be reduced from `staffSpace * 1.5` to `staffSpace * 0.75` to prevent misidentifying clicks in dense semitone clusters.
* **Absolute Coordinate Math:** The Marquee `isDragging` state must strictly track absolute `e.clientX / e.clientY` viewport coordinates during MouseDown and MouseMove. The conversion to Canvas-Center relative coordinates MUST ONLY happen mathematically at the exact moment of `onMouseUp`.

* **Objective:** Fix the detached marquee box by aligning the coordinate spaces.
* **Instructions:**
    1. In `handleMouseDown` (or the fallback in `handleCanvasMouseDown`), set the start anchor using purely absolute coordinates: 
       `marqueeStart.current = { x: e.clientX, y: e.clientY };`
    2. In `handleMouseMove`, calculate the visual rectangle using only absolute coordinates:
       ```typescript
       const x = Math.min(e.clientX, marqueeStart.current.x);
       const y = Math.min(e.clientY, marqueeStart.current.y);
       const w = Math.abs(e.clientX - marqueeStart.current.x);
       const h = Math.abs(e.clientY - marqueeStart.current.y);
       setMarqueeRect({ x, y, w, h });
       ```
    3. In `handleMouseUp`, translate the absolute `marqueeRect` into canvas-center coordinates for the intersection test:
       ```typescript
       const rect = canvasRef.current!.getBoundingClientRect();
       // Convert absolute viewport coordinates to canvas-center relative coordinates
       const mLeft = marqueeRect.x - rect.left - (rect.width / 2);
       const mRight = mLeft + marqueeRect.w;
       const mTop = marqueeRect.y - rect.top - (rect.height / 2);
       const mBottom = mTop + marqueeRect.h;
       ```
    4. Ensure the visual `marqueeRef` div remains `position: 'fixed'` so it accurately renders based on the absolute `x` and `y` properties of `marqueeRect`.
