## 1. Project Context & Objectives
The current Ottava (8va/8vb) implementation suffers from a critical rendering bug: it transposes individual notes rather than the entire staff group. This causes chords that span across the transposition threshold to invert visually (e.g., C7 renders below C6). Furthermore, the octave labels are statically anchored to the staff and overlap noteheads, rather than dynamically floating above/below the extreme notes with proper padding.

To resolve this, we must refactor the imperative rendering logic. Instead of appending DOM nodes individually on `Note On`, the engine will maintain a state of raw active notes and trigger a full `clearAndRedraw()` function. This allows the engine to calculate group-wide thresholds and apply unified transposition shifts to all notes in a given staff.

## 2. Technical Decisions & Dependencies
* **Files to Modify:** `src/components/NotationCanvas.tsx`.
* **Rendering Paradigm Shift:** `handleMidiMessage` will now only manage a `Map` of active notes (storing raw MIDI data and step offsets). It will then call `renderActiveNotes()`.
* **Group-Wide Transposition:**
  * Find the MAXIMUM `rawStepOffset` in the Treble group. If `>= 28`, shift ALL Treble notes `-14`. If `>= 21`, shift ALL Treble notes `-7`.
  * Find the MINIMUM `rawStepOffset` in the Bass group. If `<= -30`, shift ALL Bass notes `+14`. If `<= -23`, shift ALL Bass notes `+7`.
* **Dynamic Label Placement:** A single label per staff is rendered. Its Y-coordinate is anchored to the *transposed* Y-coordinate of the extreme note, plus/minus `3.5 * var(--staff-space)` for padding.
