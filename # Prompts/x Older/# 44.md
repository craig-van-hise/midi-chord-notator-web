# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
The spelling engine is currently defaulting to flat-side enharmonics for the `m7(no5)` sonority when voiced in its 3rd inversion (Pitch Class Set decimal `37`, i.e., intervals `{0, 2, 5}` from the lowest note). When this specific voicing is built with its root on the `#4` or `#5` degrees relative to the key, the default mapping falls back to flat-side roots (e.g., `Gb` and `Ab`), causing awful double-flat spellings for the rest of the chord. 

To resolve this, we must inject a deterministic override for decimal `37` to force sharp-side enharmonics (`#4` and `#5`) when the root lands on pitch class numbers `6` and `8`. 

## 2. Technical Decisions & Dependencies
* **Exception Injection:** Since decimal `37` represents a cardinality-3 set (three distinct pitch classes), this logic must be injected into the `if (cardinality > 2)` block within `getChromaticRootInterval`, rather than the `cardinality === 2` block we used previously.
* **Mathematical Solution:** By intercepting `decimal === 37` when `rootPCN` is `6` or `8`, we will explicitly return `chromaticIntervals[rootPCN][1]` (the sharp-side index). This ensures the root is spelled as `#4` or `#5` respectively, anchoring the rest of the chord extensions in clean sharp spellings (e.g., F#m7(no5) instead of Gbm7(no5)).
* **No Side Effects:** This remains a strictly confined logical update to a pure function.
