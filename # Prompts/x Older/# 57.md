# Product Requirements Prompt (PRP) — 🔘 Pending

## 1. Project Context & Objectives
The recent fix to the keyboard's bottom geometry successfully restored the `border-radius` of the white keys. However, it revealed a secondary visual artifact: the negative space created between the rounded corners of adjacent keys is exposing the parent container's white background. 

In a physical piano, the chassis beneath the keys is dark, and the gaps between keys form shadows. To restore the skeuomorphic fidelity of the virtual keyboard, we must darken the background of the parent container so that any negative space exposed by the keys' geometry renders as a natural shadow.

## 2. Technical Decisions & Dependencies
* **Chassis Shadow Color:** The parent container wrapping the `pianoKeys` array currently has `backgroundColor: '#fff'`. Because the white keys are `flex-shrink: 0`, tile perfectly, and explicitly declare their own `#fff` backgrounds, the parent's background is *only* visible through the negative space at the bottom corners. 
* **Implementation:** We will change the parent container's `backgroundColor` to a dark hex value (e.g., `#222`) to simulate the internal chassis shadow. 
* **No Side Effects:** This is a pure CSS modification localized to a single inline style object.

## 3. Implementation Phases
