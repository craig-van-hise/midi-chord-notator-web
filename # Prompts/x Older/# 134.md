# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
The previous implementation of the Autoplay fix introduced a severe regression. The global gesture listener in `App.tsx` is incorrectly re-invoking `audioEngine.init()`, which overwrites the populated `Tone.Sampler` (e.g., the Piano) with an empty dummy sampler. This causes all subsequent `noteOn` calls to throw errors, completely silencing the application.

**Objective:** 1. Fix the gesture listener in `App.tsx` to strictly execute `Tone.start()` without re-initializing the audio engine routing.
2. Remove the error-swallowing `catch` blocks in `NotationCanvas.tsx` to expose true tracebacks.
3. Verify that the PC keyboard mutations (`Alt+ArrowUp`, etc.) correctly invoke `playPreviewNotes`.

## 2. Technical Decisions & Dependencies
* **Context Unlocking:** `Tone.start()` safely resumes a suspended `AudioContext` without modifying graph routing. The `App.tsx` listener must execute this and *nothing else*.
* **Error Logging:** Hardcoded string warnings in `catch` blocks mask underlying Web Audio API errors. We must pass the raw `error` object to `console.error`.
