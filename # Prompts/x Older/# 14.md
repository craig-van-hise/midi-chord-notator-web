# Product Requirements Prompt (PRP) — ✅ Completed (With Nuclear Fix)

## 1. Project Context & Objectives
The previous UI phase failed to resolve two critical layout defects: the header title text remained drastically oversized (consuming the entire header height), and unnecessary white margins persisted on the left and right edges of the application window. Despite previous attempts, the `h1` tag seemed to be inheriting or being overridden by a massive `56px` font size.

## 2. Technical Decisions & Dependencies
* **Files Modified:** `src/index.css`, `src/App.tsx`.
* **Nuclear Font Fix:** The `h1` tag has been replaced with a `div` to bypass global `h1` selectors. Hardcoded inline styles (`style={{ fontSize: '1.125rem' }}`) have been applied to guarantee a specific, reduced size (approx 18px).
* **Global Margin Eradication:** Removed `max-width`, `margin: 0 auto`, and `padding` from `#root` in `src/index.css`.

## 3. Implementation Phases

### Phase 1: Root Layout & Global Margin Annihilation — ✅ Completed
* **Objective:** Remove the hardcoded margins/padding injected by Vite's default CSS boilerplate.
* **Result:** `src/index.css` updated. `#root`, `body`, and `html` now have `width: 100%`, `margin: 0`, and `padding: 0`.

### Phase 2: Title Typography Lockdown — ✅ Completed (Nuclear)
* **Objective:** Force the header title to a standard size (~18px) and prevent it from stretching.
* **Result:** 
    1. **Purged `src/index.css`**: Removed the `h1 { font-size: 56px }` global rule.
    2. **Refactored `src/App.tsx`**: Changed `<h1 />` to `<div />` and applied `style={{ fontSize: '1.125rem', lineHeight: '1' }}`.
    3. **Alignment**: Maintained `self-center` for vertical centering.

### Phase 3: Key Signature String Sanitization — ✅ Completed
* **Objective:** Remove the parenthesized accidental counts from the `<select>` options.
* **Result:** `src/components/KeySignatureSelector.tsx` updated to display raw key names only (e.g., "C Major").
