# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
This sprint refines the Voicing-Aware PCS Rotation logic to respect user-defined spelling overrides across octaves.

**The Bug:**
Currently, when rotating an explicitly spelled chord (e.g., F# Major in C Major), the rotation engine strips the spelling lock to allow the notes to move vertically on the staff. Because the lock is stripped, the notes are excluded from the `overrides` dictionary, causing the chord speller to revert them to their default theoretical spellings (e.g., Gb Major).

**The Solution:**
The `isLockedSpelling` bypass is an obsolete artifact. The new Override Injection Bridge is robust enough to force the correct spelling and staff position naturally. We will strip the bypass logic and instead manage the lifecycle of the `spellingOverride` string itself: Rotation will preserve it, while Diatonic and Chromatic transpositions will delete it.

## 2. Technical Decisions & Dependencies
* **Data Flow:** You will remove all references to `isLockedSpelling`. The system will rely purely on the presence of `spellingOverride` on the `ActiveNoteData` object.
* **Transposition Lifecycle:**
    * **Rotation:** `spellingOverride` remains intact (Pitch Class is unchanged).
    * **Diatonic/Chromatic:** `spellingOverride` is explicitly set to `undefined` (Pitch Class fundamentally changes).
