# Product Requirements Prompt (PRP) — 🔘 Pending

## 1. Project Context & Objectives
**Objective:** Execute immediate UX/UI hardening on the newly integrated ROMPler collapsible footer within the Grand Staff MIDI Notator app.
**Current State:** The initial integration successfully injected the ROMPler, but layout regressions have occurred. The collapse tab is vertically clipped and horizontally misaligned, the main layout lacks a scrollable viewport (causing the open ROMPler to obscure the keyboard), and the vertical gap between the notation canvas and the keyboard is excessively large.
**End State:** A perfectly responsive, scrollable main viewport that allows the user to access the keyboard even when the ROMPler is open. The ROMPler tab will be strictly aligned to the left edge and fully visible when collapsed. The keyboard and notation canvas will be grouped tightly together.

## 2. Technical Decisions & Dependencies
* **Framer Motion Math (`calc`):** The worker agent MUST NOT rely on arbitrary percentages for the closed state. The closed animation state MUST use `calc(100% - 32px)` (assuming a 32px tab height) to guarantee the tab is perfectly flush with the bottom of the viewport when closed.
* **Viewport Scrolling Architecture:** The main wrapper (excluding the top navbar/header) MUST be converted to a `flex-1 overflow-y-auto` container. It MUST feature a substantial `padding-bottom` (e.g., `pb-[400px]`) so that the user can scroll to the absolute bottom of the keyboard and clear the absolute/fixed positioning of the opened ROMPler.
* **Flexbox Proximity:** The layout container holding the Notation Canvas and the Keyboard MUST be switched to a strict `flex-col justify-start` structure with an explicit `gap-[15px]`. All intermediate `margin-top`, `flex-1`, or space-distributing utilities that artificially force the keyboard down MUST be stripped.
