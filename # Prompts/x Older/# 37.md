## 1. Project Context & Objectives
The MIDI Chord Notator is currently failing to render chord extensions (9ths, 11ths, 13ths) correctly. The interval projection logic in `chordSpeller.ts` is explicitly clamped to diatonic degrees 1-7 via strict Regex matching (`/^([b#x]*)([1-7])$/`). When the Pitch Class Set (PCS) Look-Up Table returns a compound interval (e.g., "9", "#11", "b13"), the regex fails, defaulting to `[1, 0]` (the root). This causes all extensions to be misspelled as the root note.

This PRP will guide the deterministic mathematical refactoring of the interval parsing engine to support compound intervals, map them to their simple diatonic equivalents for array lookups, and successfully return the correct note spelling.

## 2. Technical Decisions & Dependencies
* **Regex Overhaul:** All interval regex matchers must be updated from `([1-7])` to `(\d+)` to capture multi-digit compound intervals.
* **Modulo Math for Simple Degrees:** A compound degree (like 9, 11, or 13) must be reduced to its simple diatonic degree (1-7) for array lookups (`majorSteps`, `DIATONIC_NAMES`). 
    * **Formula:** `const simpleDegree = ((degree - 1) % 7) + 1;`
* **Pure Functions Only:** The fixes are contained entirely within `src/utils/chordSpeller.ts`. No changes to `notationMath.ts` are required for this phase, as `notationMath.ts` relies on the calculated spelling output.
