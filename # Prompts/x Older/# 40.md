# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
In the previous phase, we successfully routed isolated Perfect 4ths (Pitch Class Set decimal `33`) to use flat-side enharmonics. However, an edge case exception is required: when the root of the Perfect 4th falls exactly on the tritone degree relative to the key (Pitch Class Number `6`), it must use the sharp-side spelling. 

For example, in C Major, a Perfect 4th built on pitch class 6 (MIDI notes `[66, 71]`) should be spelled as `[F#, B]` rather than `[Gb, Cb]`. This PRP implements a deterministic override for this specific mathematical condition.

## 2. Technical Decisions & Dependencies
* **Exception Injection:** The `getChromaticRootInterval` function processes enharmonics based on the relative pitch class number (`rootPCN`). We will inject an explicit `if` condition before the `check14679e` array lookup to intercept `decimal === 33` when `rootPCN === 6`.
* **Mathematical Solution:** By returning `chromaticIntervals[rootPCN][1]` (the sharp-side index) for this specific conjunction, we ensure the root is spelled as `#4` (e.g., F# in C Major) instead of `b5` (Gb), allowing the perfect fourth above it to spell cleanly as `B` rather than `Cb`.
* **No Side Effects:** This remains a strictly confined logical update to a pure function.
