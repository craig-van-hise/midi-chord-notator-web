# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
The UUID selection engine is structurally sound, but suffers from two UX logic bugs: 
1. **Transposition Deselection:** Transposing a note causes it to lose its selection state. This happens because the transposition mapping function is likely generating a new UUID for the note instead of preserving the existing one.
2. **Click-Away Deselection:** Clicking on the empty background of the canvas does not deselect currently selected notes. 

The objective of this micro-sprint is to patch these two specific logic gaps in the event handlers.

## 2. Technical Decisions & Dependencies
* **ID Preservation:** During any transposition or mutation (Shift+Up, Option+Down, etc.), the `note.id` property MUST be carried over exactly as it was. New UUIDs should ONLY be generated for brand new physical notes entering the system.
* **Canvas Deselection Logic:** In the `onPointerDown` handler, if the hit-testing math determines that NO note was intersected, `selectedNoteIds.current.clear()` must be called to drop the selection.
* **Zero Layout Changes:** You are strictly forbidden from modifying the JSX layout, CSS, or rendering math.

### Phase 1: Fix Transposition ID Preservation — ✅ Completed
* **Objective:** Ensure transposed notes retain their UUIDs so they remain selected.
* **Instructions:**

### Phase 2: Implement Click-Away Deselection — ✅ Completed
* **Objective:** Allow users to clear their selection by clicking the empty canvas.
* **Instructions:**
    1. In `NotationCanvas.tsx`, locate the `onPointerDown` handler attached to the Glass Panel.
    2. Look at the section where `clickY` (or `clickX/Y`) is calculated and mathematically checked against the `activeNotes`.

    5. After the loop, add the fallback logic:

       ```javascript
           // Only clear if the user isn't holding Shift/Cmd, or clear regardless based on standard UX

               selectedNoteIds.current.clear();
           }
       }
       ```
    6. Ensure the visual highlight state updates to reflect the cleared Set (either via your imperative class toggling or by triggering a render if necessary).
