## 1. Project Context & Objectives
**Project:** Grand Staff MIDI Notator
**Current State:** Starting fresh from a reverted state. Previous AI attempts created a "split-brain" regression where live MIDI inputs and UI transformations became decoupled, causing the visual canvas and audio engine to track different notes. 
**Objective:** Implement a strictly unified, non-destructive MIDI Note Filter. The main UI keyboard, the notation canvas, the Tone.js audio engine, and the Filter Modal's 128-key UI **MUST ALL SHARE THE EXACT SAME STATE (`safeNotes`)**. 

**CRITICAL RULE:** You are strictly forbidden from creating new note data interfaces. You must use standard `number[]` arrays. You are strictly forbidden from having separate state dispatchers for live input vs. transformations.

## 2. Technical Decisions & Dependencies
* **Strict Modulo-12 Math:** `midiProcessing.ts` must use exact octave shifts (+/- 12) for wrapping to preserve pitch classes.
* **Global Hard Limit:** A final clamp of `0-127` must be applied before the filter returns the array to prevent system crashes.
* **The Unified Funnel Architecture:** You must create a single central dispatch function in the core MIDI/State handler (e.g., `MIDIProvider.tsx`). ALL inputs (live MIDI, transformations) calculate their `theoreticalNotes` and pass them into this single funnel.

## 3. Implementation Phases

### Phase 1: Pure Mathematical Filter (`midiProcessing.ts`) — 🔘 Pending
* **Objective:** Fix the math in the filter to use strict octaves and enforce global bounds.
* **Instructions:**
  1. Open `src/lib/midiProcessing.ts`.
  2. Rewrite `applyNoteFilter(notes: number[], mode: FilterMode, range: [number, number]): number[]`.
  3. **Wrap Logic:** Use strict octave shifting (`+/- 12`):
     ```typescript
     let n = note;
     if (range[1] - range[0] >= 11) { // Only attempt wrap if an octave fits
       while (n < range[0]) n += 12;
       while (n > range[1]) n -= 12;
       if (n >= range[0] && n <= range[1]) result.push(n);
     }
     ```
  4. **Block Logic:** If outside the range, do not push to the result.
  5. **Global Hard Limit:** Right before the final return, forcefully strip any notes outside `0-127`:
     `const safeBounds = result.filter(n => n >= 0 && n <= 127);`
  6. Return `Array.from(new Set(safeBounds))`.

