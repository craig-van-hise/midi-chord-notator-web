# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
An objective code audit has revealed the causes of two visual artifacts:
1. **Sticky Ledger Lines:** The `NotationCanvas` is pushing duplicate ledger lines into the render array for notes situated in the spaces above/below the staff. This creates duplicate React keys and stranded DOM nodes.
2. **False Selection Highlights:** The `Keyboard` component was hardcoded to color all refreshed notes red (`#ef4444`), making it appear as though the keyboard was poorly syncing with the notation selection state.

## 2. Technical Decisions & Dependencies
* **Ledger Line Math:** The existing `for` loops with `ls += 2` natively handle all required ledger lines perfectly. The subsequent `if` checks that push `finalStep - 1` or `+ 1` are creating duplicates and must be removed.
* **Keyboard Styling:** The red selection indicator is exclusively reserved for the `NotationCanvas`. The `Keyboard` must only ever display active notes in the standard purple accent color (`#aa3bff`).
