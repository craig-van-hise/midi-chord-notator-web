# Product Requirements Prompt (PRP) — 🔘 Pending

## 1. Project Context & Objectives
The Dual-Column architecture successfully isolated the chromatic cluster into two stacks and correctly applied the internal zipper logic to the Right Stack. However, the dynamic gap calculation failed to provide adequate clearance. The left-sweeping accidentals of the Right Stack (e.g., the natural signs for D and E) are visibly colliding with the noteheads of the Left Stack.

The objective is to refine the collision math to guarantee a rigid bounding box that explicitly separates the rightmost edge of the Left Stack from the leftmost edge of the Right Stack's accidentals. Additionally, we will introduce a minor compaction parameter to bring the accidentals slightly closer to their parent noteheads for a cleaner engraving style.

## 2. Technical Decisions & Dependencies
**Rigid Bounding Box Math:**
The previous gap calculation likely failed to account for notes in the Left Stack that were *also* zippered to the right (e.g., Eb offset over Db). 
We must calculate the absolute `Rightmost_LeftStack_Edge` by evaluating the calculated `xOffset` of *every* note in the left stack, not just the base $X=0$ coordinate.

**The Corrected Gap Formula:**
1. Let `LeftMaxX` = The maximum value of `(note.xOffset + NOTEHEAD_WIDTH)` across all notes in the `leftStack`.
2. Let `MaxRightAccidentalWidth` = The standard SMuFL width of an accidental (e.g., $\approx 1.2 \times \text{staff-space}$) if any note in the `rightStack` requires one.
3. Let `Padding` = $0.2 \times \text{staff-space}$ (minimum visual breathing room).
4. `RightStack_BaseX = LeftMaxX + MaxRightAccidentalWidth + Padding`.

**Accidental Compaction:**
To address the user's request to "compact things," the accidentals for the Right Stack should anchor slightly closer to the notehead.
* `RightAccidental_X = ParentNote_X - ACCIDENTAL_WIDTH + COMPACTION_OFFSET` (where `COMPACTION_OFFSET` is a small positive nudge, e.g., $0.15 \times \text{staff-space}$).
