# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
The application is experiencing a fatal race condition and silent keyboard transformations due to React closure traps and Web Audio API suspensions. 
1. If a user plays hardware MIDI before unlocking the `AudioContext`, the suspended audio engine receives the events, corrupts its internal buffers, and crashes 50% of the time upon subsequent unlocking.
2. PC Keyboard transformations (`Alt+ArrowUp/Down`) are successfully transposing notation but remain completely silent, likely due to stale closures in the `keydown` event listener.

**Objective:** 1. Implement a blocking "Click to Start" overlay to explicitly unlock the audio engine, resolving the UX ambiguity and preventing the 50/50 buffer crash.
2. Add a strict guard to drop all MIDI playback requests if the context is suspended.
3. Hardwire the imported `audioEngine` singleton directly into the mutation functions, bypassing React state/closures entirely.

## 2. Technical Decisions & Dependencies
* **Explicit Gatekeeper UI:** Instead of relying on invisible global listeners, `NotationCanvas.tsx` will feature an absolute-positioned overlay that requires a user click. This click will execute `await Tone.start()`.
* **The Bouncer:** `handleMidiMessage` will check `Tone.context.state === 'running'` before passing *any* note to `audioEngine.noteOn`.
* **Direct Audio Plumbing:** The `applyChromaticShift`, `applyDiatonicShift`, and `applyPcsRotation` functions will invoke `audioEngine.noteOn` directly, rather than relying on the component-scoped `playPreviewNotes` function which may be trapped in a stale closure.

### Phase 2: Hardwiring PC Keyboard Audio — ✅ Completed
* **Objective:** Ensure PC keyboard shortcuts deterministically play audio by bypassing React closures.
* **Instructions:**
    1. Open `src/components/NotationCanvas.tsx`.
    2. Locate `applyChromaticShift`, `applyDiatonicShift`, and `applyPcsRotation`.
    3. At the very end of **each** of these three functions, insert the following direct, imperative playback logic:
       ```typescript
       // Force note-offs for active previews to prevent smearing
       try { audioEngine.releaseAll(); } catch(e) {}
       
       // Play the newly mutated pitches directly
       const transposedStrings = Array.from(selectedNoteIds.current)
         .map(id => activeNotes.current.find(n => n.id === id)?.note)
         .filter((n): n is number => typeof n === 'number')
         .map(pitch => Tone.Frequency(pitch, "midi").toNote());

       transposedStrings.forEach(noteStr => {
           if (Tone.context.state === 'running') {
               try { audioEngine.noteOn(noteStr, 100 / 127); } catch (e) { console.error(e); }
               
               // Auto-release after 500ms
               setTimeout(() => {
                   try { audioEngine.releaseNote(noteStr); } catch (e) {}
               }, 500);
           }
       });
       ```
