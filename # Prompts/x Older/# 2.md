# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
While Clef/Bracket positioning and the fundamental Notehead Y-axis alignment (C=C) are now correct, the dynamic Ledger Line rendering engine in `NotationCanvas.tsx` is mathematically misaligned. Notes above the staff (e.g., C6) render lines ~2 Note Space Units (NSU) too low. Middle C (C4) renders its line incorrectly above the bass clef, and notes in the gap (like B3) are missing lines. Additionally, the application header remains significantly oversized.

This phase must correct the header scaling and overhaul the mathematical loop governing imperative ledger line generation.

## 2. Technical Decisions & Dependencies
* **Files to Modify:** `src/App.tsx`, `src/components/NotationCanvas.tsx`.
* **Metric Standard (NSU):** `staffSpace` (derived from `var(--staff-space)` CSS).
* **Ledger Correction Constant:** Based on visual audit, ledger lines require a hard upward offset (subtracting Y in screen coordinates). We define the standard correction as `(2 * staffSpace) + 3` pixels.
* **Anchoring Rule:** Ledger lines must always be mathematically anchored to the calculated `y` coordinate of the note head, not independently calculated staff line indices, to prevent Middle C drift.
