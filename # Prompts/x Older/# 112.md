# Product Requirements Prompt (PRP) — 🔘 Pending

## 1. Project Context & Objectives
The Grand Staff MIDI Notator is experiencing a critical mapping regression in its spelling engine (`chordSpeller.ts` / `NotationCanvas.tsx`). During certain mutation sequences (e.g., using rotation via `Cmd+Alt+Up`), notes in an inversion are being assigned the exact same pitch letter (e.g., an E half-diminished chord renders as "E E E E"). 

**CRITICAL CONTEXT:** This feature worked flawlessly in a recent build. The current spelling failure is a **regression**, not a fundamental architectural missing piece. The bug likely stems from a recent change in how `activeNotes` are mapped, sorted, or passed into `getChordSpelling`, or a minor logic tweak inside the speller itself that severed the correct PC-to-interval relationship.

**Objective:**
Identify the specific Git commit that introduced this regression, isolate the flawed logic, and revert or patch the logic so that continuous mutation operations correctly spell inverted chords.

## 2. Technical Decisions & Dependencies
* **Git Bisect / Diffing:** The primary investigative tool is `git log` and `git diff`.
* **No Blind Rewrites:** Do not rewrite the `PCS_LUT` logic or the `getRootSpellingFromKey` algorithm. The math was already solved in a previous commit.

## 3. Implementation Phases

* **Objective:** Find the exact commit that broke the chord spelling during rotation/transposition.
* **Instructions:**
