# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
The application's rendering engine and port state management are now stable. We must now address edge-case pitches that generate an unreadable cluster of ledger lines. When a user plays notes significantly out of the standard staff range (e.g., above C6 or below C2), the engine must apply an automatic mathematical octave transposition and attach standard Ottava notation (8va, 15ma, 8vb, 15mb). 

## 2. Technical Decisions & Dependencies
* **Target Files:** `src/components/NotationCanvas.tsx`.
* **Transposition Cutoffs (Based on Diatonic `stepOffset` from Middle C):**
    * `>= 21` (C7 and above): Transpose down 2 octaves (-14 steps). Label: **15ma**
    * `>= 14` (C6 and above): Transpose down 1 octave (-7 steps). Label: **8va**
    * `<= -14` (C2 and below): Transpose up 1 octave (+7 steps). Label: **8vb**
    * `<= -21` (C1 and below): Transpose up 2 octaves (+14 steps). Label: **15mb**
* **Stateless Rendering Paradigm:** Because the engine imperatively renders notes in real-time, the octave label will be generated per-note as a child of the `noteContainer`. To prevent visual clutter when a chord triggers multiple shifts, the label's Y-coordinate will be fixed relative to the staff boundaries, allowing duplicate labels in a chord to stack perfectly on top of one another.

## 3. Implementation Phases

### Phase 1: Mathematical Transposition Engine — ✅ Completed
* **Objective:** Intercept the raw `stepOffset` and apply mathematical shifts before the Y-coordinate is calculated.
* **Instructions (Target: `src/components/NotationCanvas.tsx` -> `isNoteOn`):**
    1. Immediately after retrieving `const { stepOffset: rawStepOffset, accidental } = getEnharmonicSpelling(...)`, establish the transposition variables:
       ```javascript
       let stepOffset = rawStepOffset;
       let octaveLabel = null;
       let isOttavaAlta = true; // true for above treble, false for below bass
       
       if (rawStepOffset >= 21) {
           stepOffset -= 14;
           octaveLabel = "15ma";
       } else if (rawStepOffset >= 14) {
           stepOffset -= 7;
           octaveLabel = "8va";
       } else if (rawStepOffset <= -21) {
           stepOffset += 14;
           octaveLabel = "15mb";
           isOttavaAlta = false;
       } else if (rawStepOffset <= -14) {
           stepOffset += 7;
           octaveLabel = "8vb";
           isOttavaAlta = false;
       }
       ```
    2. Continue using the modified `stepOffset` for all subsequent `y` calculations, ledger line logic, and zipper clustering logic.

### Phase 2: Imperative Label Generation — ✅ Completed
* **Objective:** Inject the "8va/15ma" text element into the DOM when a shift occurs.
* **Instructions (Target: `src/components/NotationCanvas.tsx` -> `isNoteOn`):**
    1. After creating the `noteContainer` and its children (notehead, accidental, ledger lines), check if `octaveLabel` is truthy.
    2. If true, generate a `<div>`:
       ```javascript
       const labelEl = document.createElement('div');
       labelEl.textContent = octaveLabel;
       labelEl.className = "ottava-label absolute font-serif italic text-black dark:text-gray-300 pointer-events-none";
       labelEl.style.fontSize = `calc(var(--staff-space) * 2)`;
       labelEl.style.left = '50%';
       labelEl.style.transform = 'translateX(-50%)';
       ```
    3. **Fixed Y-Anchoring:** We must anchor the label high above (or far below) the staff, *counteracting* the `noteContainer`'s specific Y position so all labels in a chord align horizontally:

       * If `!isOttavaAlta`: `labelEl.style.top = calc(-1 * (50% - ${y}px) + var(--staff-space) * 10)`
       *(Note: Adjust the multiplier `8` or `10` as necessary to ensure it clears the top/bottom of the visual staff staves).*
    4. Append `labelEl` to `noteContainer`.
