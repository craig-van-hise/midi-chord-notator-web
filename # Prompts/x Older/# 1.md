## 1. Project Context & Objectives
We are rectifying coordinate mapping failures and UI layout issues in the `Grand Staff Notator` (to be renamed `Chord Notator`). The underlying MIDI communication via native Web MIDI and the CustomEvent bridge (`MIDI_MESSAGE_RECEIVED`) is functioning correctly. However, the `NotationCanvas.tsx` implementation violates spacing metrics for SMuFL glyphs and fails to dynamically render ledger lines for out-of-bounds diatonic pitches.

## 2. Technical Decisions & Dependencies
* **Target Files:** `src/App.tsx`, `src/components/NotationCanvas.tsx`.
* **Metric Standard (NSU):** `NSU` (Note Space Unit) is explicitly mapped to the CSS variable `var(--staff-space)`. In JS calculations, `staffSpace` represents 1 NSU. 1 Diatonic Step equals `staffSpace / 2`.
* **Middle C Anchor:** As dictated by `.agent/rules/02-smufl-engine.md`, Middle C is the mathematical anchor (Y=0, centered).
* **Coordinate Fix:** The user noted notes render 1 NSU too low. We will apply a `+ staffSpace` correction to the vertical `y` calculation in the imperative render block.
* **Ledger Line Math:** * Middle C (stepOffset: 0, mapped Y: `0`) always requires a ledger line.
    * Notes above the treble staff (Y >= `staffSpace * 6`) require lines spaced at `staffSpace` intervals.
    * Notes below the bass staff (Y <= `-staffSpace * 6`) require lines spaced at `staffSpace` intervals.
