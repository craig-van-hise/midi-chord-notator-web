# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
The application is suffering from two severe data-pipeline bugs. 
1. **ID Destruction (Crossover Bug):** Transposing a note past another note causes the visual selection to jump to the stationary note. This happens because the `chordSpeller` or sorting pipeline is stripping the `id` from the note objects, causing the renderer to fall back on array indices.
2. **Clef Inversion (Ottava Bug):** When a wide interval exists between the staves, the Intelligent Ottava Transposition applies shifts backward (placing the treble note in the bass clef an octave lower, and vice versa). This indicates staff assignment is relying on array order instead of absolute MIDI pitch.

## 2. Technical Decisions & Dependencies
* **Data Preservation:** Any function that spells, sorts, or maps the `activeNotes` array MUST preserve the `id` string of every note.
* **Absolute Layout Math:** Staff assignment (Treble vs Bass) and Ottava shifts (8va/8vb) MUST be calculated using the absolute `midiPitch` of the individual note, never its index in the array.
* **No UX/UI Changes:** Do not change how the Marquee works, do not change the pointer events, and do not modify the CSS classes.
