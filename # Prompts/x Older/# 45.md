# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
We need to implement two broad, sweeping patterns for chromatic root note spelling based on chord types, utilizing a zero-overhead $O(1)$ lookup at runtime. 

1. **The Minor Pattern:** Any chord containing `"m"` in the `chord_type` must follow the minor triad enharmonic routing: `#1, b3, #4, #5, b7` (e.g., C#m, Ebm, F#m, G#m, Bbm in C Major).
2. **The Dominant/7th Pattern:** Any chord with `base_7th == 7`, OR any chord with `"7"` and `"no5"` but NOT `"m"` in the `chord_type`, must follow this routing: `b2, b3, #4, b6, b7` (e.g., Db7, Eb7, F#7, Ab7, Bb7 in C Major).

To ensure strict $O(1)$ performance during the hot render loop, the worker agent will first write a temporary script to extract the exact `decimal` indices matching these exact string rules from the master JSON. Those raw numbers will then be hardcoded into `chordSpeller.ts` as `Set<number>` constants.

## 2. Technical Decisions & Dependencies
* **Interface Update:** `base_7th: number;` must be added to the `PCS_Entry` interface in `src/utils/chordSpeller.ts`.
* **Extraction Logic (Run Once):**
  * `isMinor`: `entry.chord_type.includes("m")`
  * `isDominant`: `entry.base_7th === 7 || (entry.chord_type.includes("7") && entry.chord_type.includes("no5") && !entry.chord_type.includes("m"))`
* **Performance:** Hardcoded `Set<number>` constants guarantee instant evaluation without any string manipulation during the high-frequency MIDI event loop.
