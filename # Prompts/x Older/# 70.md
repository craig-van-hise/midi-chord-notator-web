## 1. Project Context & Objectives
This is a corrective sprint for the Note Selection and Editing Engine. The previous implementation suffered from SMuFL hitbox inaccuracies, DOM event delegation failures, local state desync from the global `MIDIProvider`, and a destructive Pitch Class Set (PCS) rotation algorithm that squashed open voicings. The objective is to patch these specific failures using precise mathematical center-point collision, global state synchronization, and a Voicing-Aware PCS Rotation algorithm.

## 2. Technical Decisions & Dependencies
* **Hitbox Math (Center-Point Collision):** Do NOT use AABB rectangle intersection for the marquee. You must calculate the geometric center of the note's bounding box `(x + width/2, y + height/2)`. A note is selected if and only if its *center point* falls within the Marquee's bounding box.
* **Pointer Events:** Ensure the rendered note DOM elements have `pointer-events: bounding-box` or `pointer-events: all` applied in CSS/inline styles to prevent parent containers from swallowing the click events.
* **Global State Sync:** Any transposition or rotation mutations MUST update the global state (e.g., via `MIDIProvider` context or a passed `onNotesChanged` prop) so that the `Keyboard.tsx` component instantly reflects the mutated notes.
* **Voicing-Aware Rotation Algorithm:** You are mandated to use the exact mapping algorithm provided in Phase 4 to preserve octave spreads during rotation.

### Phase 4: Voicing-Aware PCS Rotation — 🔘 Pending
* **Objective:** Implement Command + Option + Up/Down to rotate the pitch class set while preserving the original octave spread.
* **Instructions:**
    1.  You MUST use the following logic to prevent octave squashing.
    2.  **Algorithm (Rotate UP):**
        ```javascript
        // 1. Get unique pitch classes sorted ascending
        const pcs = Array.from(new Set(selectedNotes.map(n => n % 12))).sort((a,b)=>a-b);
        
        // 2. Map each selected note to the NEXT pitch class in its relative register
        const rotatedNotes = selectedNotes.map(note => {
            const currentPC = note % 12;
            const currentPcsIndex = pcs.indexOf(currentPC);
            const nextPcsIndex = (currentPcsIndex + 1) % pcs.length;
            const targetPC = pcs[nextPcsIndex];
            
            // 3. Find the nearest targetPC strictly ABOVE the current note
            let newNote = note;
            while(newNote % 12 !== targetPC) { newNote++; }
            return newNote;
        });
        ```
    3.  **Algorithm (Rotate DOWN):** Same as above, but `nextPcsIndex = (currentPcsIndex - 1 + pcs.length) % pcs.length;` and the `while` loop decrements `newNote--` until it hits the `targetPC`.
    4.  Update the global state with `rotatedNotes`.
