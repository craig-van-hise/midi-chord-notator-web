# Product Requirements Prompt (PRP) — 🔘 Pending

## 1. Project Context & Objectives
This sprint expands the `NotationCanvas` into a fully functional editing environment. The user must be able to visually write notes onto the staves using a new "Write Mode", delete selected notes, and seamlessly undo/redo their actions. 

Crucially, because `NotationCanvas` acts as the source of truth for computational mutations, every action (Write, Delete, Undo, Redo) must broadcast its final state back to the `MIDIProvider` to ensure the `Keyboard` component stays in perfect sync.

## 2. Technical Decisions & Dependencies
* **Undo/Redo Architecture:** To avoid React rendering lag during high-frequency MIDI events, we will manage the history stacks via mutable `useRef` arrays (`ActiveNoteData[][]`). State is only pushed to history on discrete user interactions (Delete, Transpose, Write, Mouse-Drop), *not* on continuous MIDI hardware input.
* **Write Mode Rendering (Imperative):** The translucent "ghost note" must follow the mouse at 60fps. Do not use React state (`useState`) for its coordinates. You must define a static `#ghost-note` DOM element and manipulate its `style.top` and `style.display` imperatively in the `onPointerMove` handler.
* **Y-Axis Snapping Math:** The visual grand staff has a non-linear vertical gap between the Treble and Bass staves. I have reverse-engineered the snapping formula. You must use the exact coordinate-to-step mapping logic provided in Phase 3.

### Phase 3: Write Mode Interactivity — 🔘 Pending
* **Objective:** Map mouse movements to diatonic steps and write notes on click.
* **Instructions:**
  1. In `handlePointerMove`, add logic that only fires if `isWriteMode.current === true`:
     * Calculate `const pointerY = e.clientY - rect.top;`
     * **The Snapping Math:**
       ```typescript
       const canvasCenterY = rect.height / 2;
       const relativeY = canvasCenterY - pointerY;
       let stepOffset = 0;
       
       // Handle the visual gap between Treble and Bass staves
       if (relativeY >= 0) {
           stepOffset = Math.round((relativeY - staffSpace) / (staffSpace / 2));
       } else {
           stepOffset = Math.round((relativeY + staffSpace) / (staffSpace / 2));
       }
       
       // Calculate snapped Y position
       const snappedY = canvasCenterY - (((stepOffset) * (staffSpace / 2)) + (relativeY >= 0 ? staffSpace : -staffSpace));
       
       const ghost = document.getElementById('ghost-note');
       if (ghost) {
           ghost.classList.remove('hidden');
           ghost.style.top = `${snappedY}px`;
           ghost.dataset.step = stepOffset.toString();
       }
       ```
  2. In `handlePointerDown`:
     * If `isWriteMode.current === true`:
       * Check bounds to see if they clicked "away". The staves are `300px` wide.
       * `const pointerX = e.clientX - rect.left;`
       * `const isOutsideStaves = pointerX < (rect.width / 2 - 150) || pointerX > (rect.width / 2 + 150);`
       * If `isOutsideStaves`, set `isWriteMode.current = false`, hide ghost note, and `return;`.
       * Otherwise, they clicked *on* the measure. Call `commitState()`.
       * Read the step offset: `const ghost = document.getElementById('ghost-note'); const step = parseInt(ghost?.dataset.step || '0');`
       * Calculate the pitch: `const newPitch = transposeDiatonically(0, step, keySignatureRef.current);` (Middle C is index 0).
       * Create a new note object with `generateId()`, push to `activeNotes.current`, call `updateSpellings()`, and `updateActiveNotes([...activeNotes.current])`.
       * *Crucial Check:* Do NOT fall through to the selection/marquee logic if a note was just written. Return early.
