# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
The spelling engine is currently defaulting to sharp-side enharmonics for all isolated minor 3rd sonorities (Pitch Class Set decimal `9`, i.e., `{0, 3}`). However, when a minor 3rd is built on the `b3` or `b7` degrees relative to the current key, music theory dictates these should be spelled using flat-side enharmonics. 

For example, in C Major, playing MIDI notes `[63, 66]` (built on the `b3` degree, `rootPCN` 3) currently spells as `[D#, F#]`. It should be spelled as `[Eb, Gb]`. Similarly, `[70, 73]` (built on the `b7` degree, `rootPCN` 10) should be spelled as `[Bb, Db]` instead of `[A#, C#]`. This PRP implements a deterministic override for these specific mathematical conditions.

## 2. Technical Decisions & Dependencies
* **Exception Injection:** We will inject another explicit `if` condition within `getChromaticRootInterval` to intercept `decimal === 9` specifically when the relative pitch class number (`rootPCN`) is either `3` or `10`.
* **Mathematical Solution:** By returning `chromaticIntervals[rootPCN][0]` (the flat-side index) for this conjunction, we ensure the root is spelled as `b3` or `b7` respectively, allowing the minor 3rd above it to spell cleanly with flats rather than sharps.
* **No Side Effects:** This remains a strictly confined logical update to a pure function.
