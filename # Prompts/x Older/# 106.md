## 1. Project Context & Objectives
**Objective:** Enhance the MIDI playback engine and UI synchronization between the Notation Canvas and the Virtual Keyboard.
**Current State:** The virtual keyboard currently sends MIDI events unconditionally on both pointer down (Note On) and pointer up (Note Off), which triggers audio regardless of toggle state. The notation canvas is completely silent when notes are added or selected. The virtual keyboard displays purple for actively playing notes but is completely blind to the red selection state managed by the Notation Canvas.
**End State:** 1. The virtual keyboard strictly limits click-driven `audioEngine` playback to "Note On" events (toggling). 
2. The `NotationCanvas` triggers a highly precise 500ms `audioEngine` preview (at velocity 85) whenever a new note is placed on the staff or an existing note is selected. 
3. The `Piano88` component visually maps the red `#ef4444` highlight used by the Notation Canvas to physically selected keys, enforcing unidirectional data flow from the canvas down to the keyboard.

## 2. Technical Decisions & Dependencies
* **Engine Injection:** We are injecting the `audioEngine` directly into `NotationCanvas.tsx`. The conversion requires `Tone.Frequency(midiNote, "midi").toNote()` to format the string payload for the engine.
* **Preview Normalization:** The requested velocity of 85 maps to `0.67` (85/127) for the Tone.js/smplr engine. The timeout MUST be exactly `500ms`. We must use a `useRef` to track active timeouts and clear them if the component unmounts.
* **Selection State Propagation:** The `NotationCanvas` currently hides its selection state (`selectedNoteIds.current`) completely inside its component closure. To pass this to the keyboard, the worker agent MUST lift this state up to the `useMidi` provider, adding `selectedNotes: number[]` to the global store, so `Piano88` can subscribe to it without prop-drilling through the main layout tree.
* **Selection Highlight Priority:** In `Piano88`, the standard active color is `#aa3bff` (purple). If a key exists in the new `selectedNotes` array from the global context, it MUST override the purple and display the exact selection red: `#ef4444`.
