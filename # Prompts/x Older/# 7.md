# Product Requirements Prompt (PRP) — 🔘 Pending

## 1. Project Context & Objectives
The notation rendering and collision algorithms are now visually stable. However, the core music theory engine (`src/utils/notationMath.ts`) suffers from a critical semantic failure: it does not properly handle diatonic spelling for "white key" enharmonics (Cb, Fb, B#, E#). 

Currently, in `Gb Major`, MIDI note 59 (Cb4) incorrectly spells as B3, assigning it the wrong `stepOffset` and accidental. This occurs because the engine naively maps MIDI pitch classes to static step values without adjusting the mathematical octave boundary (MIDI octaves split between B/C, but diatonic spelling shifts depending on the key signature).

This phase will completely rewrite `getEnharmonicSpelling` to use a deterministic Diatonic Scale Generator algorithm and introduce an octave-boundary correction matrix.

## 2. Technical Decisions & Dependencies
* **Files to Modify:** `src/utils/notationMath.ts`.
* **Algorithmic Approach:** We will abandon hardcoded semitone maps. Instead, we will generate the diatonic pitch classes for the active key signature by applying mathematical transformations to a base C-Major array using the Order of Sharps/Flats.
* **Octave Boundary Correction:** MIDI pitch 59 is mathematically Octave 3. However, if it is spelled as **Cb4**, it must be treated as Octave 4 to render on the correct staff line. Conversely, MIDI 60 is mathematically Octave 4, but if spelled as **B#3**, it must be treated as Octave 3.

## 3. Implementation Phases

### Phase 1: The Diatonic Scale Generator — 🔘 Pending
* **Objective:** Implement a deterministic function that returns the 7 correct pitch classes and accidentals for any given key signature.
* **Instructions (Target: `src/utils/notationMath.ts`):**
    1. Define the core arrays outside the function:
       ```typescript
       const C_MAJOR_SCALE = [
         { step: 0, pc: 0 }, { step: 1, pc: 2 }, { step: 2, pc: 4 }, { step: 3, pc: 5 },
         { step: 4, pc: 7 }, { step: 5, pc: 9 }, { step: 6, pc: 11 }
       ];
       const ORDER_OF_SHARPS = [3, 0, 4, 1, 5, 2, 6]; // F, C, G, D, A, E, B (Steps)
       const ORDER_OF_FLATS  = [6, 2, 5, 1, 4, 0, 3]; // B, E, A, D, G, C, F (Steps)
       const KEY_SIG_ACCIDENTALS: Record<string, number> = {
         "C Major": 0, "G Major": 1, "D Major": 2, "A Major": 3, "E Major": 4, "B Major": 5, "F# Major": 6, "C# Major": 7,
         "F Major": -1, "Bb Major": -2, "Eb Major": -3, "Ab Major": -4, "Db Major": -5, "Gb Major": -6, "Cb Major": -7
       };
       ```
    2. Create `getDiatonicMap(keySignature: string)`:
       * Initialize a deep copy of `C_MAJOR_SCALE` adding an `acc: string | null = null` property to each object.
       * Lookup the accidental count from `KEY_SIG_ACCIDENTALS`.
       * **If Sharps (> 0):** Loop `i` from 0 to `count`. Get `step = ORDER_OF_SHARPS[i]`. Increment the `pc` of that step by 1: `scale[step].pc = (scale[step].pc + 1) % 12`. Set `scale[step].acc = '#'`.
       * **If Flats (< 0):** Loop `i` from 0 to `Math.abs(count)`. Get `step = ORDER_OF_FLATS[i]`. Decrement `pc` by 1: `scale[step].pc = (scale[step].pc + 11) % 12`. Set `scale[step].acc = 'b'`.
       * Return a `Map<number, { step: number, acc: string | null }>` where the key is the transformed `pc`.

### Phase 2: Octave Correction & Step Calculation — 🔘 Pending
* **Objective:** Rewrite `getEnharmonicSpelling` to utilize the map and apply mathematical corrections for Cb and B#.
* **Instructions (Target: `src/utils/notationMath.ts`):**
    1. Inside `getEnharmonicSpelling(note, keySignature)`:
       ```typescript
       const pitchClass = note % 12;
       const baseOctave = Math.floor(note / 12) - 1; // Standard MIDI octave
       const diatonicMap = getDiatonicMap(keySignature);
       ```
    2. Look up the `pitchClass` in `diatonicMap`.
    3. **If Diatonic (found in map):**
       * Retrieve `step` and `acc`.
       * Apply **Octave Correction:**
         * `let diatonicOctave = baseOctave;`
         * `if (pitchClass === 11 && step === 0) diatonicOctave += 1;` // Cb correction (MIDI B but diatonic C)
         * `if (pitchClass === 0 && step === 6) diatonicOctave -= 1;` // B# correction (MIDI C but diatonic B)
       * Calculate final position: `const stepOffset = ((diatonicOctave - 4) * 7) + step;`
       * Return `{ stepOffset, accidental: acc }`.
