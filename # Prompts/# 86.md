# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
The application is currently suffering from a "Keyboard Blackout" bug. When a note is transposed in the canvas, the global state is updated with an array of `ActiveNoteData` objects. However, the `Keyboard.tsx` component is failing to render the highlighted keys because it expects an array of raw integers (MIDI numbers) and its `.includes()` check fails. The objective is to patch the `Keyboard` component to handle object-based active notes so the virtual piano accurately reflects the canvas state.

## 2. Technical Decisions & Dependencies
* **Robust Type Checking:** `Keyboard.tsx` must be updated to handle `activeNotes` whether it contains raw numbers (from legacy MIDI input) or objects (from the Canvas state).
* **Data Preservation:** We MUST continue passing the full objects from `NotationCanvas` to `updateActiveNotes`. You are strictly forbidden from flattening the array into numbers, as this will destroy the UUIDs and break the selection engine.
