
# Product Requirements Prompt (PRP) — 🔘 Pending

## 1. Project Context & Objectives
The Grand Staff MIDI Notator currently supports a "Toggle Mode" for displaying chords. We must introduce a new "Hold Mode" feature alongside it. This feature mimics an intelligent sustain pedal: it accumulates and holds active notes on the display while the mode is enabled. When the user lifts all fingers from the MIDI controller (`numKeysDown === 0`), the chord remains rendered on the canvas. However, the system enters a "ready to clear" state, whereby the *very next* MIDI NoteOn event immediately flushes the previously held chord and begins a fresh pitch set collection. Additionally, existing UI elements require a copy update for conciseness.

## 2. Technical Decisions & Dependencies
**State Machine Architecture:**
Because the application uses an imperative engine for high-frequency MIDI events to bypass React's render cycle, the Hold Mode logic must be implemented in the core event consumer/accumulator (likely within the layer subscribing to `MIDI_MESSAGE_RECEIVED`).

The agent must implement the following state machine variables and logic within the notation engine:

* **State Variables:**
    * `physicalKeysDown`: A `Set<number>` tracking currently depressed MIDI pitch numbers.
    * `activeDisplayNotes`: A `Set<number>` tracking pitches to be rendered.
    * `isHoldModeEnabled`: A `boolean` (syncs with React UI state via a custom event or ref).
    * `isWaitingForNewChord`: A `boolean` initialized to `false`.

* **Algorithmic Flow:**
    * **On NoteOn(pitch):**
        1. `physicalKeysDown.add(pitch)`
        2. IF `isHoldModeEnabled` AND `isWaitingForNewChord` === `true`:
            * `activeDisplayNotes.clear()`
            * `isWaitingForNewChord = false`
        3. `activeDisplayNotes.add(pitch)`
        4. Trigger Engine Redraw with `activeDisplayNotes`.
    * **On NoteOff(pitch):**
        1. `physicalKeysDown.delete(pitch)`
        2. IF `!isHoldModeEnabled`:
            * `activeDisplayNotes.delete(pitch)`
        3. IF `isHoldModeEnabled` AND `physicalKeysDown.size === 0`:
            * `isWaitingForNewChord = true`
        4. Trigger Engine Redraw with `activeDisplayNotes`.
    * **On Hold Mode Toggled OFF:**
        1. Set `isHoldModeEnabled = false`
        2. `isWaitingForNewChord = false`
        3. Sync display: `activeDisplayNotes = new Set(physicalKeysDown)`
        4. Trigger Engine Redraw with `activeDisplayNotes`.
