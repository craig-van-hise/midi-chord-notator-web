# Product Requirements Prompt (PRP) — 🔘 Pending

## 1. Project Context & Objectives
**Critical Bug Fix Operation:** The application's note selection architecture is suffering from severe CSS Box Model overlap collisions. The `fontSize` required to render the SMuFL `\uE0A4` glyph creates an invisible `line-height` bounding box over 60px tall. Because notes are rendered 6px apart vertically, these massive invisible boxes overlap, causing clicks to be intercepted by adjacent notes. Additionally, the Marquee selection is misfiring on simple clicks because it lacks a minimum drag threshold.

## 2. Technical Decisions & Dependencies
* **Hitbox Geometric Containment:** The `div` wrapping the SMuFL text node MUST be explicitly constrained to `width: calc(var(--staff-space) * 1.5)` and `height: calc(var(--staff-space) * 1.5)`. The text must be centered within it using `display: flex; align-items: center; justify-content: center; line-height: 0;`. This shrinks the clickable `pointer-events: auto` area to perfectly match the visual notehead, eliminating all overlap.
* **Marquee Drag Threshold:** A mouse click (down and up) naturally carries 1-3 pixels of accidental movement. The `handleMouseUp` function must immediately return if `marqueeRect.w < 5 && marqueeRect.h < 5` to prevent it from executing intersection math and wiping the selection on a standard click.
* **Marquee Math Extraction:** Do not use `getBoundingClientRect()` on the DOM `marqueeRef` node during `onMouseUp`, as React state batching makes it unreliable. Calculate the selection bounds directly from the `marqueeRect` state object (`left: marqueeRect.x`, `right: marqueeRect.x + marqueeRect.w`, etc.).
