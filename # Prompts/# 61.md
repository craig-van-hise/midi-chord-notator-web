# Product Requirements Prompt (PRP) — 🔘 Pending

## 1. Project Context & Objectives
**Bug Fix:** The user cannot select individual notes via click or marquee, and transposition key commands are incorrectly applying to the entire chord instead of a localized selection.
**Root Cause:** The `#notes-layer` div in `NotationCanvas.tsx` implements `pointer-events-none`, rendering all child note containers unclickable. Furthermore, visual feedback for selection is missing, and the transposition logic is overly permissive when the selection set is empty.

## 2. Technical Decisions & Dependencies
* **Pointer Events:** The `.notation-note-container` must explicitly declare `pointer-events-auto` or `pointer-events: auto` in its style/classes to receive standard click events.
* **Selection Highlighting:** Selected notes must be visually unmistakable. Apply `color: '#ffaa3b'` (or a high-contrast theme color) and an aggressive drop-shadow when a note's pitch exists in the `selectedPitches` set.
* **Strict Scoping:** Transposition functions must **only** iterate over `selectedPitches`. If `selectedPitches` is empty, the key commands should do absolutely nothing. (Do not fall back to modifying the whole chord).

## 3. Implementation Phases

### Phase 1: Unblock Hit Testing & Visual Feedback — 🔘 Pending
* **Objective:** Allow notes to receive clicks and visually broadcast their selected state.
* **Instructions:**
    1. In `NotationCanvas.tsx`, locate the `.notation-note-container` mapping.
    2. Add `pointer-events: 'auto'` to its inline `style` object.
    3. Modify the notehead text node's inline `style`. Add a condition: `const isSelected = selectedPitches.has(note.note);`. If `isSelected` is true, set the `color` to a highly visible warning/accent color (e.g., `#ff9900` or `#eab308`) and apply `textShadow: '0 0 12px rgba(255, 153, 0, 0.6)'`.
    4. Ensure the `onClick` handler correctly updates `selectedPitches` (incorporating standard click, Cmd/Ctrl click, and Shift click as previously defined).
* **TDD Checkpoint:** * **Test Case 1:** Render the canvas with a note. Query the note container and assert it has `pointer-events: auto` in its computed style.
    * **AGENT INSTRUCTION:** Set Phase status to 🔵 Active. Apply fixes and verify the DOM attribute updates. Once PASS, set status to ✅ Completed.

### Phase 2: Strict Transposition Scoping — 🔘 Pending
* **Objective:** Ensure transposition shortcuts only mutate selected notes.
* **Instructions:**
    1. In the `keydown` event listener for transpositions (Up/Down arrows + modifiers), immediately check `if (selectedPitches.size === 0) return;`.
    2. Ensure the mutation loop *only* processes the pitches currently inside `selectedPitches`. Unselected pitches in `activeNotes` must remain untouched.
    3. When the mutation completes, update `selectedPitches` to contain the *new* transposed pitches so the user doesn't lose their selection upon moving it.
* **TDD Checkpoint:** * **Test Case 1:** Given `activeNotes` [60, 64, 67] and `selectedPitches` [64]. Trigger `Shift + ArrowUp`. Assert `activeNotes` becomes [60, 65, 67] and `selectedPitches` updates to [65].

### Phase 3: Marquee Intersection Fix — 🔘 Pending
* **Objective:** Ensure the marquee drag correctly populates `selectedPitches`.
* **Instructions:**
    1. In the `onMouseUp` handler of the marquee logic, verify the math. Use `element.getBoundingClientRect()` for the marquee box and for each `.notation-note-container`.
    2. A standard 2D AABB (Axis-Aligned Bounding Box) intersection test must be used:
       `!(rect2.left > rect1.right || rect2.right < rect1.left || rect2.top > rect1.bottom || rect2.bottom < rect1.top)`
    3. If they intersect, parse the `data-midi-note` and add to the selection set.
