# Product Requirements Prompt (PRP) — 🔘 Pending

## 1. Project Context & Objectives
The previous implementation of the "Click to Start" overlay completely failed to render in the physical browser due to CSS/DOM obscuration and React state misconfigurations. The agent relied on headless tests that passed, while the visual component was hidden from the user. 

Furthermore, relying on `Tone.context.state` or a raw React state variable inside the `useEffect` MIDI and Keyboard listeners causes stale closure bugs, resulting in silent failures.

**Objective:** 1. Implement a literal, explicitly styled, absolute-positioned JSX overlay that cannot be hidden.
2. Implement a `useRef` alongside `useState` to safely track the audio unlock state inside `useEffect` closures without triggering infinite re-renders or stale data traps.
3. Block `audioEngine` calls natively using this Ref.

## 2. Technical Decisions & Dependencies
* **Dual State Tracking:** We will use `const [isAudioUnlocked, setIsAudioUnlocked] = useState(false);` for the UI rendering, AND `const isAudioUnlockedRef = useRef(false);` for the event listeners. They will be updated synchronously.
* **Top-Level DOM Placement:** The overlay MUST be the very first child of the `<div ref={canvasRef} className="notation-canvas-container...">` block, equipped with `z-[9999]` and `backdrop-blur` to guarantee visibility.
