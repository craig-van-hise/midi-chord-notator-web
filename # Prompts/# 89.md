# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
An objective code audit of `NotationCanvas.tsx` has exposed fatal flaws in the Shift-Click (Range Selection) logic. The code is attempting to parse a UUID string as an integer, resulting in `NaN` math, which causes the selection to fail and triggers accidental "micro-marquee" selections. Furthermore, the code is overwriting the selection anchor unconditionally on every click, destroying the ability to "scale back" a range selection. The objective is to patch the range selection logic to accurately look up pitches by UUID and correctly preserve the anchor.

## 2. Technical Decisions & Dependencies
* **Pitch Lookup:** You are strictly forbidden from using `.split('-')` or string parsing to derive pitch from an ID. You must use an array `.find()` against `renderedNotes` to retrieve the anchor note's absolute `note` value.
* **Anchor Preservation:** `lastSelectedNoteId.current` acts as the selection anchor. It MUST ONLY be updated during Single Clicks and Cmd/Ctrl Clicks. It must NEVER be updated during a Shift-Click.
* **Scale-Back Support:** To allow a user to scale a selection back, the Shift-Click logic must call `selectedNoteIds.current.clear()` before running the bounds loop.

## 3. Implementation Phases

### Phase 1: Patch Shift-Click Logic — ✅ Completed
* **Objective:** Fix the `NaN` bug and implement true anchor-based range selection.
* **Instructions:**
    1. Open `src/components/NotationCanvas.tsx` and locate the `if (e.shiftKey && lastSelectedNoteId.current)` block inside `handlePointerDown`.
    2. **Delete** the string splitting logic:
       ```javascript
       const [lastPitchStr] = lastSelectedNoteId.current.split('-');
       const lastPitch = parseInt(lastPitchStr);
       ```
    3. Replace it with a robust array lookup:
       ```javascript
       const anchorNote = renderedNotes.find(n => n.id === lastSelectedNoteId.current);

       const min = Math.min(pitch, lastPitch);
       const max = Math.max(pitch, lastPitch);
       ```
    4. Immediately after calculating `min` and `max`, clear the selection to support scaling back:
