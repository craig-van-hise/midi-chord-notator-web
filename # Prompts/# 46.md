# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
The spelling engine is currently defaulting to sharp-side enharmonics for isolated minor 6th sonorities (Pitch Class Set decimal `257`, i.e., intervals `{0, 8}`). When a minor 6th is built on the `b7` degree relative to the current key (`rootPCN === 10`), it must be spelled using flat-side enharmonics. 

For example, in C Major, playing MIDI notes `[70, 78]` (built on the `b7` degree) currently defaults to sharp spellings (`[A#, F#]`). It must be corrected to spell as `[Bb, Gb]`. This PRP injects a deterministic override for this specific mathematical condition.

## 2. Technical Decisions & Dependencies
* **Exception Injection:** Since decimal `257` represents a cardinality-2 set, this logic will be injected into the existing `if (cardinality === 2)` block within `getChromaticRootInterval`.
* **Mathematical Solution:** By intercepting `decimal === 257` specifically when `rootPCN === 10`, we will return `chromaticIntervals[rootPCN][0]` (the flat-side index). This ensures the root is correctly anchored as a flat (`Bb`), forcing the interval above it to follow suit (`Gb`).
* **No Side Effects:** This remains a strictly confined logical update to a pure function.
