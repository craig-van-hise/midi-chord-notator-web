# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
The current notation engine successfully implements a basic "zipper pattern" for 2-note clusters. However, it fails to correctly render dense cohemitonia (clusters of three or more adjacent semitones or seconds). According to standard engraving rules (e.g., Dorico), when a third overlapping note is introduced, it must cascade further to the right. 

We must upgrade the layout engine in `notationMath.ts` to utilize a dynamic, level-based collision avoidance algorithm.

## 2. Technical Decisions & Dependencies
**Algorithmic Shift:**
The agent is strictly forbidden from using a binary `isLeft`/`isRight` toggle for cluster offsets. You must implement a **Greedy X-Level Assignment** algorithm.

**The Math / Algorithm:**
1. Let `COLLISION_THRESHOLD = 1` (measured in diatonic staff steps, where 1 equals the distance between a line and an adjacent space).
2. Create an array of arrays tracking placed notes at each horizontal tier: `const xLevels: NotePosition[][] = []`.
3. Sort the active chord notes by pitch (lowest to highest).
4. Iterate through the sorted notes:
   * Initialize `currentLevel = 0`.
   * **Condition:** Check if the current note's Y-position (diatonic steps) collides with *any* note already placed in `xLevels[currentLevel]`. A collision is defined as `Math.abs(placedNote.ySteps - currentNote.ySteps) <= COLLISION_THRESHOLD`.
   * If a collision exists, increment `currentLevel` by 1 and check again.
   * If no collision exists, place the note in `xLevels[currentLevel]` and assign `note.xLevel = currentLevel`.
5. The final CSS/SVG offset for rendering will be `xLevel * NOTEHEAD_WIDTH` (e.g., `xLevel * 1.18em` for SMuFL standard noteheads).
