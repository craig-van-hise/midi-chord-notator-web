## 1. Project Context & Objectives
**Architectural Rewrite:** The browser's native DOM event system cannot handle click hit-testing for dense musical clusters due to unavoidable geometric overlapping of SMuFL glyphs. 
We are abandoning DOM-level `onClick` handlers on individual notes. We will implement a unified, mathematics-based spatial hit-testing engine at the root `<div className="notation-canvas-container">` level to intercept all mouse events, calculate distances using the Pythagorean theorem, and execute selection logic deterministically.

## 2. Technical Decisions & Dependencies
* **Spatial Coordinate Mapping:** The canvas origin $(0,0)$ for our notes is the exact center of the `.notation-canvas-container`.
* **Distance Formula:** To find the clicked note, use $d = \sqrt{(x_{mouse} - x_{note})^2 + (y_{mouse} - y_{note})^2}$.
* **Bypass React Closures:** To avoid stale state in mouse handlers, create a `selectionStateRef` that always mirrors `selectedPitches`, and a `renderedNotesRef` that always mirrors `renderedNotes`. The mouse handlers will read from these refs, make decisions, and then update the React state.
