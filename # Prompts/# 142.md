## 1. Project Context & Objectives
**Project:** Grand Staff MIDI Notator
**Current State:** The system has functional MIDI mappings, velocity/duration tracking, and a home anchor system. However, mathematical transformations can push MIDI notes out of the valid playable range (<0 or >127), causing the Tone.js/smplr engine to crash or hang.
**Objective:** Integrate the user-provided `MidiNoteRangeFilter.tsx` component to act as a safety gate and creative constraint between the transformation output and the audio engine. The filter must be accessible via a new button (with a keyboard icon) in the global settings/transformations menu, open in a dedicated modal, and apply its mathematical bounding logic to the audio payload *before* playback.

## 2. Technical Decisions & Dependencies
* **Data Flow Architecture:** The filter applies *exclusively* to the audio output payload. The visual notation engine should continue receiving the raw, mathematically transformed note arrays (unless the visual notation engine natively crashes on out-of-bounds, in which case the filter must be applied before visual state update. For this phase, apply to Audio only).
* **State Management:** Introduce `outputFilterMode` (type `FilterMode`, default: `'block'`) and `outputFilterRange` (type `[number, number]`, default: `[0, 127]`) to `usePersistentState.ts`.
* **Mathematical Filter Algorithms (`src/lib/midiProcessing.ts`):** You must implement the following exact algorithm to process the note arrays safely. Note the infinite-loop guard on `octave_wrap`.

```typescript
export type FilterMode = 'block' | 'octave_wrap' | 'wrap' | 'limit';

export const applyNoteFilter = (notes: number[], mode: FilterMode, range: [number, number]): number[] => {
  const [min, max] = range;
  const span = max - min + 1;
  const result: number[] = [];

  for (let n of notes) {
    if (n >= min && n <= max) {
      result.push(n);
      continue;
    }

    switch (mode) {
      case 'block':
        break; // drop note
      case 'limit':
        result.push(Math.max(min, Math.min(max, n)));
        break;
      case 'octave_wrap':
        let octN = n;
        // Guard against infinite loops if range is smaller than 1 octave
        if (max - min >= 11) { 
          while (octN < min) octN += 12;
          while (octN > max) octN -= 12;
          if (octN >= min && octN <= max) result.push(octN);
        }
        break;
      case 'wrap':
        const wrapped = min + (((n - min) % span) + span) % span;
        result.push(wrapped);
        break;
    }
  }
  // Deduplicate notes that resolved to the same pitch
  return Array.from(new Set(result));
};

```
