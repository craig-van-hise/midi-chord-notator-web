# Product Requirements Prompt (PRP) — 🔘 Pending

## 1. Project Context & Objectives
The current layout engine successfully tiers notes based on diatonic 2nd intervals ($yStep$ distance $\le 1$), but it critically fails when handling **chromatic unisons** (multiple notes sharing the exact same diatonic staff line/space, e.g., $Db$ and $D\natural$). 

According to standard engraving rules (as observed in Dorico), when a pitch set contains multiple variations of the same note letter:
1. The notes must be split into two distinct visual columns.
2. The flatter version is placed in the left column.
3. The sharper version is placed in the right column, pushed far enough right to avoid the left column.
4. The sharper version **must** explicitly display its accidental (e.g., a natural sign), even if the key signature would normally suppress it.

We must upgrade the topological tagging (`assignXLevels`) and the React render layer to detect shared $ySteps$, force the rightward shift, and force the rendering of the explicit accidental.

## 2. Technical Decisions & Dependencies
**The "Chromatic Split" Algorithm:**
During Pass 1 (`assignXLevels`), the agent must group notes by their diatonic $yStep$.
* If a $yStep$ contains only one note, it is processed through the standard L0/L1 zipper logic.
* If a $yStep$ contains multiple notes (e.g., MIDI 61 $Db$ and MIDI 62 $D\natural$ both sitting on the D space):
    1. Sort them from flattest (lowest MIDI pitch) to sharpest.
    2. The flatter note is assigned to the standard zipper (L0 or L1 depending on surrounding notes).
    3. The sharper note is explicitly assigned to a new "Right Block" (`xLevel = 2`).
    4. The sharper note is mutated to include a new boolean property: `forceAccidentalDisplay: true`.

**Explicit Accidental Enforcement:**
In the rendering layer, if `forceAccidentalDisplay === true`, the component MUST render the SMuFL accidental glyph corresponding to the note's exact spelling (e.g., fetching the Natural glyph `U+E261` if the note is a D natural), bypassing any "hide natural" logic normally applied by the key signature.
