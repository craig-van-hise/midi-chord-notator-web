## 1. Project Context & Objectives
**Objective:** Integrate a standalone React/Tone.js MIDI ROMPler into the existing "Grand Staff MIDI Notator" web application. 
**Current State:** The ROMPler is a standalone full-screen application with its own Web MIDI instance (`useMidi.ts`). The Notator application already possesses a global OMNI MIDI input engine.
**End State:** The ROMPler will operate as a collapsible, slide-up footer at the bottom of the Notator app. It will strictly consume MIDI events from the Notator's existing OMNI engine, stripping away its redundant Web MIDI implementation.

## 2. Technical Decisions & Dependencies
* **Dependency Migration:** The worker agent MUST install the following ROMPler dependencies into the host Notator application: `tone@^15.1.22`, `smplr@^0.20.0`, `lucide-react`, `motion`, `clsx`, `tailwind-merge`.
* **Architectural Decoupling:** The `src/audio/useMidi.ts` file from the ROMPler will be **deleted/ignored**. The Notator's existing global MIDI message listener will be modified to broadcast events directly to the ROMPler's `audioEngine`.
* **Payload Translation:** The ROMPler's engine expects note values as strings (e.g., "C4"). The Notator's MIDI handler must convert raw MIDI note numbers (0-127) using `Tone.Frequency(data1, "midi").toNote()` before invoking `audioEngine.noteOn()` and `audioEngine.releaseNote()`.
* **Collapsible UI Paradigm:** The `App.tsx` from the ROMPler will be refactored into `RomplerFooter.tsx`. The `min-h-screen bg-[#0c0c0d]` wrapper must be replaced with a `fixed bottom-0 left-0 w-full z-50` container, using Framer Motion (`<motion.div>`) to slide up/down via a toggle tab residing just above the footer.
