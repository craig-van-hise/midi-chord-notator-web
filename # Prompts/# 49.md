# Product Requirements Prompt (PRP) — 🔵 Local Validation Complete

## 1. Project Context & Objectives
The application is failing to deploy via GitHub Actions due to three distinct CI/CD build errors. These are strictness errors enforced by TypeScript (`verbatimModuleSyntax`), a missing property on a fallback object stemming from our recent interface update (`base_7th`), and a deprecation warning from GitHub runners regarding Node.js 20. 

This PRP outlines the exact, file-by-file fixes required to clear the build queue and successfully deploy the app.

## 2. Technical Decisions & Dependencies
* **Type-Only Imports:** Vite and modern TypeScript configurations often enforce `verbatimModuleSyntax: true`. This means any imported identifier used *only* as a type must be explicitly imported using the `type` keyword to allow the bundler to safely strip it out.
* **Interface Synchronization:** When we added `base_7th: number;` to the `PCS_Entry` interface, any hardcoded mock or fallback objects returning `PCS_Entry` immediately became invalid. We will patch the specific fallback object in `binaryLut.ts` causing the type failure.
* **CI/CD Node Versioning:** GitHub Actions will deprecate Node 20 runners. We will preemptively bump the deployment pipeline to Node 24.
