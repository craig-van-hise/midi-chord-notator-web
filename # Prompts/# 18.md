# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
The previous phase successfully implemented the Ottava (8va/8vb) rendering logic and fixed-Y text anchoring. However, the threshold bounds were set far too aggressively (C6 and C2), which clutters the UI with unnecessary transposition labels for notes that advanced musicians naturally read on ledger lines. 

We must immediately widen the natural notation boundaries. The application must render natural ledger lines up to B6 on the top, and down to Bb0 on the bottom. Transposition should ONLY trigger at the extreme ends of the 88-key grand piano range: C7 and above, or A0 and below.

## 2. Technical Decisions & Dependencies
* **Files to Modify:** `src/components/NotationCanvas.tsx`.
* **Middle C (C4) Anchor Reference:** `stepOffset === 0`.
* **New Mathematical Thresholds:**
    * **C7** is exactly 3 octaves above Middle C: `stepOffset = 21`.
    * **C8** is 4 octaves above: `stepOffset = 28`.
    * **C1** is 3 octaves below: `stepOffset = -21`.
    * **A0** is a third below C1: `stepOffset = -23`.
* **Logic Override:** Replace the existing threshold numbers in the `isNoteOn` render block with these exact boundaries.

## 3. Implementation Phases

### Phase 1: Widen Transposition Bounds — ✅ Completed

       ```javascript
           // A0 up to C0 (technically A0, Bb0, B0)
           stepOffset += 7;
           octaveLabel = "8vb";
           isOttavaAlta = false;
       }
       // Anything between stepOffset -22 (B0) and 20 (B6) remains naturally notated.
       ```
