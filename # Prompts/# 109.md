# Product Requirements Prompt (PRP) — 🔘 Pending

## 1. Project Context & Objectives
**Objective:** Integrate the `NavController` UI from `midi-nav-controller-web` into the MIDI Chord Notator as a collapsible, 50% scaled tab appended to the bottom of the virtual keyboard card.
**Current State:** The virtual keyboard rests in a standard layout wrapper (`App.tsx`). Transposition and rotation algorithms exist solely as keyboard event handlers inside the `NotationCanvas.tsx` closure.
**End State:** A new `NavFooter` component will house the imported `NavController`, scaling it down to 50% via CSS transforms. This tab will slide out from underneath the keyboard card. Clicking its D-Pad buttons will dispatch `CustomEvent` payloads that replicate the exact algorithms (semitone transposition, voicing rotation) already implemented for `ArrowKey` shortcuts. The "Play" button will serve as an audition switch with dynamic velocity mapped to the Y-axis click coordinate, and the "Home" button will trigger an Undo event to revert to the initial chord state.

## 2. Technical Decisions & Dependencies
* **DOM Placement:** To ensure the Nav Controller slides out directly from the *Keyboard Card*, the worker agent MUST modify the keyboard's wrapper div in `App.tsx` to be `relative` and place the new `NavFooter` container *inside* it with absolute positioning (`absolute top-full left-1/2 -translate-x-1/2 z-0`). 
* **Scaling Paradigm:** The worker agent MUST use `transform: scale(0.5)` with `transform-origin: top center` on the imported `NavController` to shrink the UI without breaking its internal flexbox spacing.
* **Event Bridging:** Because `NotationCanvas` holds the transposition logic in its private closure, the `NavController` MUST dispatch global events: 
    * `window.dispatchEvent(new CustomEvent('NOTATOR_COMMAND', { detail: { action: 'TRANSPOSE', delta: 1 } }))`
    * `window.dispatchEvent(new CustomEvent('NOTATOR_COMMAND', { detail: { action: 'ROTATE', delta: 1 } }))`
    * `window.dispatchEvent(new CustomEvent('NOTATOR_COMMAND', { detail: { action: 'UNDO' } }))`
    * `window.dispatchEvent(new CustomEvent('NOTATOR_COMMAND', { detail: { action: 'AUDITION_PLAY', velocity: calculatedVelocity } }))`
    * `window.dispatchEvent(new CustomEvent('NOTATOR_COMMAND', { detail: { action: 'AUDITION_STOP' } }))`

### Phase 2: Event Bridging — 🔘 Pending
* **Objective:** Wire the `NavController` button events to dispatch standard custom events.
* **Instructions:**
  1. In `NavFooter.tsx`, implement the `onButtonDown` handler for the `NavController`.
  2. Map the D-Pad:
      * `UP`: Dispatch `'NOTATOR_COMMAND'`, `action: 'TRANSPOSE', delta: 1`.
      * `DOWN`: Dispatch `'NOTATOR_COMMAND'`, `action: 'TRANSPOSE', delta: -1`.
      * `RIGHT`: Dispatch `'NOTATOR_COMMAND'`, `action: 'ROTATE', delta: 1`.
      * `LEFT`: Dispatch `'NOTATOR_COMMAND'`, `action: 'ROTATE', delta: -1`.
      * `HOME`: Dispatch `'NOTATOR_COMMAND'`, `action: 'UNDO'`.
  3. Map the Play Button with Y-Axis Velocity:
      * For the `PLAY` button down event, intercept the native mouse/touch event to calculate the relative Y position.
      * Calculation: `let vel = Math.floor((offsetY / elementHeight) * 127); vel = Math.max(1, Math.min(127, vel));` (Note: ensure 0 is top and 127 is bottom if reversing logic is needed, align with standard MIDI implementations where bottom strike is often harder/higher velocity).
      * Dispatch `'NOTATOR_COMMAND'`, `action: 'AUDITION_PLAY', velocity: vel`.
  4. In `onButtonUp`, if the ID is `PLAY`, dispatch `'NOTATOR_COMMAND'`, `action: 'AUDITION_STOP'`.
