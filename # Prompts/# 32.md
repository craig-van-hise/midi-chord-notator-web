# Product Requirements Prompt (PRP) — 🔘 Pending

## 1. Project Context & Objectives
The previous implementation successfully split chromatic unisons into left and right groups, but it failed structurally in two ways:
1. It did not apply the standard "zipper" logic (alternating horizontal offsets for 2nd intervals) to the newly created right-side notes.
2. It pushed the accidentals for the right-side notes all the way to the far-left of the entire chord structure, causing massive legibility issues.

We must upgrade the layout engine to treat dense clusters as two distinct, independent columns. The engine must apply baseline zippering to both columns independently, and inject the right-side accidentals safely *between* the two columns.

## 2. Technical Decisions & Dependencies
**Dual-Column Architecture:**
You will abandon the linear `xLevel` sequence and implement a `ColumnBlock` abstraction.
1. **Pass 1 (Logical Split):** Notes are sorted by pitch and split into `leftBlock` (flatter/standard notes) and `rightBlock` (sharper chromatic unison notes).
2. **Pass 2 (Independent Zippering):** Run the previously restored baseline 2nd-interval zippering algorithm on the `leftBlock` in isolation. Then, run that exact same baseline algorithm on the `rightBlock` in isolation.
3. **Pass 3 (Dynamic Gap Calculation):** Calculate the starting horizontal anchor for the `rightBlock` dynamically:
   * `RightBlockBaseX = Max(LeftBlock X-Offsets) + NOTEHEAD_WIDTH + Max(RightBlock Accidental Widths) + PADDING`.
   * Add `RightBlockBaseX` to every offset calculated for the `rightBlock` in Pass 2.
4. **Pass 4 (Accidental Placement):** Anchor accidentals strictly to their parent notehead's `xOffset` and left-sweep from there, ensuring right-stack accidentals settle in the calculated gap.
