# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
The dynamic Ottava label rendering (8va/15ma, 8vb/15mb) successfully tracks the highest and lowest notes, but the vertical padding offset is currently far too large. This excessive padding causes the Treble labels to clip against the top of the canvas container, and pushes the Bass labels dangerously low. 

We must tighten the multiplier used in the CSS `top` calculation so the text closely "hugs" the extreme noteheads, bringing the labels safely back into the visual viewport.

## 2. Technical Decisions & Dependencies
* **Target Files:** `src/components/NotationCanvas.tsx`.
* **Padding Adjustment:** The current offset from the extreme notehead is `3.5 * var(--staff-space)`. We will reduce this to a strict `1.8 * var(--staff-space)` to provide just enough clearance for the standard SMuFL notehead height without wasting vertical space.
* **Clipping Override:** To ensure no future clipping occurs on extreme bounds like C8 or A0, we will enforce `overflow-visible` on the primary staff layout containers.

## 3. Implementation Phases

### Phase 1: Tighten Label Padding — ✅ Completed

* **Instructions (Target: `src/components/NotationCanvas.tsx` -> `renderActiveNotes`):**
