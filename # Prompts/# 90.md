## 1. Project Context & Objectives
The application currently exhibits a mathematical flaw in its Diatonic Transposition logic (`Alt + ArrowUp/Down`). The existing algorithm relies on a nearest-neighbor pitch-class snap, which fails to respect enharmonic spelling and theoretical scale degrees. For example, moving an Eb up diatonically in C Major snaps it to E natural, whereas theoretically it must move to the next scale degree (F).

This sprint will replace the nearest-neighbor algorithm with a highly deterministic, staff-position (`stepOffset`) based transposition engine. By leveraging the theoretical data already assigned by the `chordSpeller` pipeline, we can map pitch classes directly to their diatonic step offsets, apply the delta, and reverse-calculate the target MIDI note.

## 2. Technical Decisions & Dependencies
* **Algorithm Mandate:** You are strictly forbidden from using "nearest pitch class" array mapping for this feature. You will extract the transposition logic into a pure, testable function in `notationMath.ts` that calculates the target MIDI note based on the `stepOffset` and octave boundary corrections.
* **Preserve Immutability:** When applying transpositions in `NotationCanvas.tsx`, you must spread the existing note object (`{ ...noteData }`) to preserve its UUID, velocity, and channel.
* **Mathematical Solution:** I have pre-calculated the bidirectional mapping logic for Octave Boundary Corrections (e.g., Cb and B# edge cases) to offload cognitive complexity. Implement the exact code provided in Phase 1.

```typescript
/**
 * Calculates the target MIDI note for a diatonic transposition.
 * * @param currentStepOffset - The current diatonic staff position of the note.
 * @param delta - The shift amount (+1 for up, -1 for down).
 * @param keySignature - The active key signature string.
 * @returns The new MIDI note, clamped between 0 and 127.
 */
export function transposeDiatonically(currentStepOffset: number, delta: number, keySignature: string): number {
    const targetStepOffset = currentStepOffset + delta;
    
    // Calculate scale step (0-6) and octave offset relative to Middle C (C4)
    const targetScaleStep = ((targetStepOffset % 7) + 7) % 7;
    const octaveOffset = Math.floor(targetStepOffset / 7);
    let targetOctave = 4 + octaveOffset;

    // Look up target Pitch Class from the active key's diatonic map
    const diatonicMap = getDiatonicMap(keySignature);
    let targetPC = 0;
    for (const [pc, data] of diatonicMap.entries()) {
        if (data.step === targetScaleStep) {
            targetPC = pc;
            break;
        }
    }

    // Reverse Octave Boundary Corrections
    if (targetPC === 11 && targetScaleStep === 0) targetOctave -= 1; // Cb correction (Cb4 -> MIDI B3)
    if (targetPC === 0 && targetScaleStep === 6) targetOctave += 1;  // B# correction (B#3 -> MIDI C4)

    // Calculate final MIDI pitch and clamp to valid range
    const newMidiNote = ((targetOctave + 1) * 12) + targetPC;
    return Math.max(0, Math.min(127, newMidiNote));
}

```
